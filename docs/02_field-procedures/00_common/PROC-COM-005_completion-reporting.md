# PROC-COM-005 完了報告

## 0. 文書管理

| 項目 | 内容 |
|---|---|
| 手順ID | `PROC-COM-005` |
| 手順名 | 完了報告 |
| 領域 | 共通 |
| 版 | 0.1 |
| 状態 | 初版草案 |
| 作成日 | 2026-07-22 |

> 本文書は清掃・設備・警備に共通する作業完了報告の標準手順である。個別物件の報告様式、契約仕様、法定様式、承認経路及び保存期間を併用する。完了報告は、作業結果を事実どおり引き渡す行為であり、管理者の結果確認、契約上の検収、顧客への提出又は異常案件の解決を自動的に意味しない。

## 1. 目的

実施した作業について、対象、実施範囲、結果、未実施、要観察、異常、残留リスク及び後続対応を、証跡とともに所定の受領者へ報告する。予定どおり終了した作業だけでなく、条件付き完了、中止及び未完了も同じ基準で記録し、作業実態と報告上の完了が乖離すること、残課題の責任が不明になること及び承認・検収との混同を防ぐ。

## 2. 業務カタログとの関係

| 区分 | 業務ID | 業務名 | 本手順との関係 |
|---|---|---|---|
| 主業務 | `BM-13-04` | 作業完了を報告する | 作業者から管理者へ結果と残課題を引き渡す |
| 関連業務 | `BM-13-01`〜`BM-13-03` | 実績、内容、写真・動画を登録する | 完了報告の根拠となる実施事実と証跡を残す |
| 関連業務 | `BM-13-05` | 作業結果を確認する | 報告後の未入力、異常値、証跡不足等を確認する |
| 関連業務 | `BM-13-06` | 作業結果を承認する | 確認済み結果を承認又は差し戻す |
| 関連業務 | `BM-13-07`〜`BM-13-10` | 報告書作成、提出、承認状況管理 | 顧客向け編集・提出・受領へ接続する |
| 関連業務 | `BM-13-11` | 異常を速報・上申する | 完了報告を待てない異常を先行して上申する |
| 関連業務 | `BM-17-03`〜`BM-17-04` | 不適合・是正措置を管理する | 不良、未完了及び再作業を後続案件へ接続する |
| 関連業務 | `BM-17-09` | 証跡・報告書を保存する | 契約・法令に従って報告記録を保管する |

## 3. 適用範囲

### 3.1 適用対象

- 清掃、設備、警備その他の計画作業、依頼作業、点検、保守、異常対応及び復旧確認
- 完了、条件付き完了、未完了又は中止となった作業
- 複数人、複数会社又は複数工程で実施し、統合報告が必要な作業
- 顧客向け報告、日報・月報、検収又は法定記録の基礎となる現場結果
- 次勤務、専門担当、管理者又は顧客判断へ残作業・要観察事項を引き渡す場面

### 3.2 適用除外・禁止事項

- 緊急事象の速報を完了報告まで遅らせること
- 未実施、基準外、証跡不足又は残留リスクを省略して「完了」と報告すること
- 作業者報告だけで、技術確認、施設利用再開、契約上の検収又は顧客承認が済んだと扱うこと
- 事実確認なしに原因、復旧、品質合格又は顧客影響なしと断定すること
- 承認された訂正手続を経ずに元記録を削除・上書きすること
- 個人情報、機密情報又は防犯情報を権限のない相手・媒体へ含めること

## 4. 実施体制

| 役割 | 主な責任 |
|---|---|
| 作業者 | 実施事実、結果、未実施、異常及び証跡を記録し、自己確認する |
| 作業責任者 | 複数作業者・工程の結果を統合し、報告内容と残課題を確認する |
| 技術・品質確認者 | 必要な場合に測定値、品質、復旧条件及び判定根拠を確認する |
| 管理者・受領者 | 報告を受領し、不足、差戻し、後続案件、担当及び期限を確定する |
| 元請け・統括者 | 再委託先の結果を検証・統合し、契約上の報告責任を果たす |
| 顧客・施設側権限者 | 契約上の検収、利用再開、追加対応又はリスク受容を判断する |

一人が複数役割を兼ねる場合も、作業者確認、技術・品質確認、管理者受領及び契約上の承認をどの役割で行ったか区別する。

## 5. 開始条件・必要資料

### 5.1 開始条件

- 作業を終了、中止又は引継ぎ可能な安全状態にした
- 実施対象、作業指示・計画ID、参照手順及び適用版を特定できる
- 実施者ごとの結果、開始・終了時刻及び未実施項目を収集できる
- 異常は必要に応じて `PROC-COM-004` により先行上申している
- 区域を継続する場合は、維持責任者、巡視条件、期限及び解除条件を引き継いでいる
- 報告先、期限、様式、必須証跡及び承認経路を確認している

### 5.2 必要資料

- 作業指示、計画、契約仕様及び物件別完了基準
- 対応手順、チェックリスト、図面、台帳及びメーカー手順
- 測定記録、作業前後・異常箇所の写真、連絡履歴及び承認記録
- 異常、不適合、是正、修繕、引継ぎ等の後続案件情報
- 顧客指定・法定の報告様式、提出期限及び保存要件

## 6. 報告単位と状態

| 状態 | 条件 | 報告上の扱い |
|---|---|---|
| 完了 | 対象工程を実施し、基準適合、必要な原状回復・記録・引渡しを満たす | 完了として報告可能 |
| 条件付き完了 | 現場工程は終了したが、要観察、軽微な残作業、承認待ち又は継続案件がある | 条件、担当、期限を明示して報告 |
| 未完了 | 必須工程、品質確認、復旧、証跡又は引渡しが不足する | 未完了理由と再実施計画を報告 |
| 中止 | 危険、設備・環境条件、指示取消し等により継続しない | 中止理由、安全状態、再開条件を報告 |

次の状態は個別に記録し、同一視しない。

1. 現場作業の終了
2. 作業者の自己確認
3. 技術・品質確認
4. 管理者による報告受領・結果確認
5. 異常・是正案件の解決
6. 技術的復旧
7. 施設利用再開
8. 契約上の検収・承認
9. 顧客・行政等への提出・受領

## 7. 標準手順

| No. | 実施内容 | 完了基準 |
|---:|---|---|
| 1 | 作業指示、対象、参照手順、予定工程及び完了基準を確認する | 報告対象の境界が明確 |
| 2 | 実施者、開始・終了時刻、実施工程、測定値及び使用資材を集約する | 実施事実を追跡できる |
| 3 | 予定と実績を照合し、未実施、変更、追加作業及び理由を記録する | 差異が隠されていない |
| 4 | 品質・正常性、原状回復、区域、利用・運転状態を確認する | 現在状態と判断者が明確 |
| 5 | 要観察、異常、残留リスク、苦情、破損及び逸脱を整理する | 正常結果と例外を分離できる |
| 6 | 後続案件ID、担当、期限、次回確認及び終了条件を設定する | 残課題の責任が空白でない |
| 7 | 必須写真、測定記録、連絡・承認履歴等の証跡を関連付ける | 報告内容を検証できる |
| 8 | 6章の状態に基づき総合結果を判定し、作業者・責任者が確認する | 完了表現と実態が一致 |
| 9 | 所定の受領者へ期限内に報告し、受領・差戻し・追加指示を記録する | 受領状態を追跡できる |
| 10 | 差戻しや訂正に対応し、元記録と変更履歴を保ったまま再提出する | 最新内容と訂正経緯が明確 |

## 8. 完了報告の必須情報

- 物件、棟、階、区画、設備、巡回点等の対象
- 作業指示・計画ID、手順ID・版、チェックリストID・版
- 実施者、所属、必要資格、確認者
- 予定及び実績の開始・終了時刻
- 実施内容、測定値、使用資材、品質・正常性判定
- 未実施、計画変更、追加作業及び理由
- 異常、要観察、一次対応、残留リスク及び影響範囲
- 技術的復旧、区域解除、施設利用・運転再開の各状態と判断者
- 写真、動画、測定記録、連絡履歴等の証跡
- 後続案件ID、担当役割、期限、次回確認及び引継ぎ先
- 総合結果、報告日時、受領者、差戻し・追加指示

## 9. 受領・確認・承認

| 状態 | 意味 | 標準対応 |
|---|---|---|
| 作成中 | 作業者が結果を編集中 | 完了報告未提出 |
| 提出済み | 所定の受領者へ送付済み | 受領確認待ち |
| 受領済み | 管理者が報告の存在を確認 | 内容確認・後続設定へ進む |
| 差戻し | 不足、不整合又は根拠不明がある | 元記録を保持して訂正・再提出 |
| 結果確認済み | 必須項目、証跡及び後続接続を確認 | 必要に応じ承認・検収へ進む |
| 承認・検収済み | 権限者が契約・品質上承認した | 提出・請求等の後工程へ進む |

本手順の完了条件は、原則として「受領済み」かつ残課題の引渡しが成立した状態までとする。「結果確認済み」「承認・検収済み」は後続業務であり、契約・物件ルールにより本手順へ組み込む場合も状態を分けて記録する。

## 10. 権限・上申

| 判断・行為 | 権限 | 標準的な扱い |
|---|---|---|
| 実施事実、測定値、写真及び未実施理由の登録 | L0 | 作業者が事実どおり記録 |
| 承認済み範囲内の軽微な記載訂正・証跡追加 | L1 | 変更履歴を残して実施 |
| 作業順、範囲、運転・利用条件の変更承認 | L2 | 所定の運用権限者が判断 |
| 契約外作業、追加費用、納期・報告条件の変更 | L3 | 契約・予算権限者が判断 |
| 重大リスク受容、長期停止、更新投資 | L4 | オーナー等が判断 |
| 技術的復旧 | 専門・操作権限 | 作業報告と分けて判断する |
| 施設利用再開 | 施設運用権限 | 技術的復旧・区域解除と分けて判断する |
| 契約上の検収・承認 | 契約上の権限 | 作業者又は再委託先が代行しない |

危険、重大異常、法令違反のおそれ又は顧客影響の拡大を認知した場合は、報告書の完成を待たず `PROC-COM-004` により上申する。

## 11. 不足・差戻し・訂正

| 事象 | 対応 |
|---|---|
| 必須項目・証跡が不足 | 不足内容、補完担当、期限を記録し、未完了又は条件付き完了とする |
| 測定値・時刻・対象が不整合 | 原記録と機器・写真等を照合し、推測で補完しない |
| 完了基準未達 | 不適合・是正案件へ接続し、完了表現を修正する |
| 異常の上申漏れ | 安全状態を確認して直ちに上申し、遅延理由を記録する |
| 提出後に誤りが判明 | 訂正理由、変更箇所、訂正者、日時を残し、受領者へ再通知する |
| 受領者が不明・不在 | 物件別の代替受領経路へ接続し、提出済みのまま放置しない |
| 再委託先の報告が不十分 | 元請けが確認・差戻しを行い、根拠なしに補完・承認しない |

## 12. 記録・完了条件

- [ ] 対象、作業指示、手順・帳票版及び実施体制を特定した
- [ ] 予定と実績、未実施、変更及び理由を記録した
- [ ] 品質・正常性、原状回復、区域及び利用・運転状態を区別した
- [ ] 異常、要観察、残留リスク及び上申実績を記録した
- [ ] 必須証跡を関連付け、欠落時の補完担当・期限を設定した
- [ ] 後続案件、担当、期限、次回確認及び終了条件を設定した
- [ ] 総合結果を「完了／条件付き完了／未完了／中止」で判定した
- [ ] 所定の受領者、報告日時、受領状態及び差戻し内容を記録した
- [ ] 訂正・再提出がある場合、元記録と変更履歴を保持した

本手順は、報告が所定の受領者に受領され、未完了・要観察・異常を含む後続事項の担当、期限及び確認条件が引き渡された時点で完了できる。結果確認、是正完了、検収及び顧客提出は別状態として追跡する。
