# PROC-COM-004 異常上申

| 項目 | 内容 |
|---|---|

| 領域 | 共通 |
| 版 | 0.1 |
| 状態 | 初版草案 |
| 作成日 | 2026-07-22 |

> 本文書は清掃・設備・警備に共通する、異常の速報、上申及び対応責任の引渡し手順である。人命保護、退避、緊急停止及び警察・消防等への通報を遅らせてはならない。個別物件の緊急連絡網、事故対応計画、消防計画、保安規程及び領域別手順を併用する。

## 1. 目的

現場で認知した異常について、安全を確保したうえで、事実、影響、一次対応、残留リスク及び必要な判断を適切な役割へ速報・上申し、受領確認と対応責任の引渡しを成立させる。単に連絡を発信した状態と、権限を持つ者が受領して次の行動を引き受けた状態を分け、報告漏れ、伝達遅延、責任の空白及び現場権限を超えた判断を防ぐ。

## 2. 業務カタログとの関係

| 区分 | 業務ID | 業務名 | 本手順との関係 |
|---|---|---|---|
| 主業務 | `BM-13-11` | 異常を速報・上申する | 異常情報と対応責任を権限者へ引き渡す |
| 関連業務 | `BM-10-01`〜`BM-10-03` | 不具合受付、緊急度判断、一次対応 | 設備等の不具合を案件化し初動へ接続する |
| 関連業務 | `BM-11-06`、`BM-11-09`〜`BM-11-10` | 事故・災害対応、緊急連絡 | 警備・防災上の緊急事象へ接続する |
| 関連業務 | `BM-12-06`〜`BM-12-08` | 状況連絡、結果報告、周知 | 顧客・利用者への連絡を権限者へ接続する |

## 3. 適用範囲

### 3.1 適用対象

- 清掃、設備、警備その他の作業中に発見した異常、危険、不適合又は要観察事項
- 警報、顧客・利用者連絡、遠隔監視、協力会社等から受け付けた異常情報
- 現場裁量を超える停止、利用制限、追加費用、契約外対応又はリスク受容の判断要求
- 一次対応後も残るリスク、原因未確定、再発可能性又は継続監視事項
- 法令、規程、契約又は緊急計画に基づく通報・報告の要否確認

### 3.2 適用除外・禁止事項

- 切迫危険時に、様式記入や上司への順次連絡を優先して退避、救護、緊急停止又は公的機関への通報を遅らせること
- 未確認情報を確定事実として伝えること
- 現場権限を超えて技術的復旧、施設利用再開、追加費用又は重大リスク受容を決定すること
- 連絡を発信しただけで上申完了又は異常解決と扱うこと
- 個人情報、映像、設備機密等を受領権限のない相手・媒体へ送ること

## 4. 実施体制

| 役割 | 主な責任 |
|---|---|
| 発見者・受付者 | 人命・安全を優先し、観察事実、時刻、場所及び発生源を記録する |
| 現場対応者 | 権限範囲内で安全確保・一次対応を行い、状態変化を継続報告する |
| 現場責任者 | 緊急度、上申先、応援、業務継続可否及び暫定措置を判断する |
| 専門責任者 | 技術判断、停止・隔離、復旧条件及び専門業者への接続を判断する |
| 契約・施設側権限者 | 利用制限、対外周知、契約外対応、費用及び施設利用再開を判断する |
| 元請け・統括者 | 再委託先等からの速報を受領し、顧客・関係者への統合報告を行う |

一人が複数役割を兼ねる場合も、どの権限で判断したかを記録する。

## 5. 開始条件・必要情報

上申開始に完全な情報が揃うことを待たない。最低限、次を特定できた時点で速報する。

- [ ] 発見・受信時刻、場所、対象
- [ ] 現在確認できている事実
- [ ] 人命、安全、建物利用、設備、顧客・第三者への影響
- [ ] 実施済みの安全確保・一次対応
- [ ] 現場に残る危険と、直ちに必要な判断・支援
- [ ] 発信者、現場連絡手段及び次回報告予定

設備ID、原因、損害額等が未確定でも、切迫性がある場合は「未確認」と明示して速報する。

## 6. 緊急度と連絡方法

| 区分 | 代表条件 | 初動・連絡 |
|---|---|---|
| 緊急 | 人命危険、火災・犯罪、重大漏洩・感電、被害拡大、重要機能停止のおそれ | 退避・救護・緊急停止等を優先し、公的機関・緊急連絡先へ即時通報。並行して責任者へ速報 |
| 高 | 利用制限、設備停止、法令・契約影響、顧客影響の拡大が見込まれる | 安全確保後、定めた即時連絡手段で責任者へ速報し判断を要請 |
| 中 | 直ちに危険ではない異常、不適合、反復する要観察 | 所定時間内に案件登録・上申し、担当・期限・再確認条件を設定 |
| 低 | 軽微で承認済み範囲内に是正済み、傾向監視が必要 | 作業記録・定期報告へ残し、再発・閾値到達時の上申条件を設定 |

物件別基準が本表より厳しい場合は物件別基準を適用する。緊急度は新情報により随時見直す。

## 7. 標準手順

| No. | 実施内容 | 完了基準 |
|---:|---|---|
| 1 | 危険源から離れる、立入を制限する、救護・緊急停止等の安全確保を行う | 人命保護と被害拡大防止を優先した状態 |
| 2 | 時刻、場所、対象、観察事実、情報源を記録する | 事実と推測を区別できる |
| 3 | 人命、利用者、設備、業務、法令及び周辺への影響を暫定評価する | 緊急度と未確認範囲が明確 |
| 4 | 権限範囲内の一次対応を実施し、実施前後の状態を記録する | 操作・措置・残留リスクが明確 |
| 5 | 物件別連絡網から、事象・時刻・権限に合う上申先と代替先を特定する | 宛先を個人名でなく役割・責任で選定 |
| 6 | 初報を発信し、必要な判断・支援と回答期限を明示する | 8章の必須情報を伝達 |
| 7 | 受領者に復唱・回答を求め、受領時刻、指示、対応責任者を記録する | 誰が次の行動を引き受けたか明確 |
| 8 | 受領未確認又は応答不能なら代替経路へエスカレーションする | 連絡不能のまま放置しない |
| 9 | 状態変化、追加事実、一次対応結果を続報する | 事実・緊急度・依頼事項が更新済み |
| 10 | 後続案件、担当、期限、次回報告時刻及び終了条件を確定する | 上申手順と異常案件の状態を分離できる |

## 8. 初報の必須情報

初報は次の順で簡潔に伝える。

1. 自分の役割、物件・場所及び連絡手段
2. 何が、いつ、どこで起きているかという確認済み事実
3. 人命・安全、利用者、設備及び業務への現在の影響
4. 実施済みの退避、停止、隔離その他の一次対応
5. 未確認事項、残留リスク及び状態変化の可能性
6. 必要な判断、応援、専門対応、周知又は通報
7. 次回報告予定時刻

原因仮説を伝える場合は、確定事実と明確に分ける。写真・映像は安全を損なわず、取扱権限を満たす場合に限り添付する。

## 9. 受領・引渡し判定

| 状態 | 条件 | 標準対応 |
|---|---|---|
| 発信済み | 初報を送信したが受領確認がない | 上申未完了。代替経路へ連絡 |
| 受領済み | 受領者が事実と依頼事項を復唱した | 指示・担当判断を確認 |
| 条件付き引渡し | 対応責任者は決まったが、追加確認・応援待ち等がある | 発見者側の監視・報告条件を明示 |
| 引渡し完了 | 対応責任者、次の行動、期限、報告条件が確定した | 上申手順を完了可能 |
| 引渡し不能 | 適格者不在、通信不能、指揮不明又は受領拒否 | 上位・代替・公的機関へ段階的に接続 |

上申の引渡し完了は、異常の解消、技術的復旧、施設利用再開又は契約上の完了を意味しない。

## 10. 権限・上申

| 判断・行為 | 権限 | 標準的な扱い |
|---|---|---|
| 事実記録、通報、承認済み一次対応 | L0 | 作業者が実施 |
| 切迫時の退避、作業中止、緊急停止 | 緊急権限 | 人命・被害拡大防止を優先し即時報告 |
| 軽微な安全確保、応急措置、再確認 | L1 | 教育・承認済み範囲内で実施 |
| 臨時運転、部分停止、区域制限、担当変更 | L2 | 現場・専門・施設責任者が判断 |
| 契約外作業、追加費用、長時間の利用制限 | L3 | 契約・予算権限者が判断 |
| 長期停止、更新投資、重大リスク受容 | L4 | オーナー等が判断 |
| 技術的復旧 | 専門・操作権限 | 資格、メーカー手順、保安規程等に従う |
| 施設利用再開 | 施設運用権限 | 技術的復旧と分けて判断する |

## 11. 連絡不能・誤報・情報更新

| 事象 | 対応 |
|---|---|
| 第一連絡先が応答しない | 物件別の待機時間・代替順序に従い、次の役割へ接続する |
| 全連絡手段が使用不能 | 現場安全を維持し、代替通信・対面連絡・公的機関を利用する |
| 初報に誤りが判明 | 訂正であること、変更箇所、判明時刻、影響を直ちに伝える。元記録を消去しない |
| 緊急度が上昇 | 新しい緊急度に対応する経路へ切り替え、既存受領者にも通知する |
| 異常が一時的に消失 | 解決と断定せず、発生事実、原因未確定及び再発条件を引き渡す |
| 指示が安全・法令・権限に反する疑い | 実施を保留し、根拠を確認して上位・専門責任者へ再上申する |

## 12. 記録・完了条件

- [ ] 発見・受信、一次対応、各連絡、受領、指示及び引渡しの時刻を記録した
- [ ] 事実、推測、未確認事項及び情報源を区別した
- [ ] 緊急度、影響範囲、状態変化及び残留リスクを記録した
- [ ] 上申先の役割、連絡手段、受領者、復唱内容及び指示を記録した
- [ ] 対応責任者、後続案件ID、次の行動、期限、次回報告時刻を記録した
- [ ] 法定・公的通報、顧客周知及び事故報告の要否と判断者を記録した
- [ ] 写真、映像、測定値等の証跡を権限・保存要件に従って登録した

本手順は、適格な受領者への引渡しが成立し、対応責任者、次の行動、期限及び報告条件が記録された時点で完了できる。異常案件は別に継続追跡する。

## 13. 領域別手順との境界

| 文書 | 正とする内容 |
|---|---|
| 本手順 | 緊急度、速報内容、連絡経路、受領確認、対応責任の引渡し |
| 清掃手順 | 汚染、破損、遺失物、清掃不良等の発見基準と安全な一次対応 |
| 設備手順 | 警報・測定・設備状態の判断、停止・隔離、技術的復旧 |
| 警備手順 | 事件・事故・災害の初動、現場保全、公的機関への接続 |
| 完了報告手順 | 作業結果、未完了事項及び顧客向け完了報告 |

領域別手順で異常を認知・一次対応した後、本手順により権限者へ接続する。領域別帳票へ本手順の必須記録を組み込む場合、重複する共通帳票は作成しなくてよい。
