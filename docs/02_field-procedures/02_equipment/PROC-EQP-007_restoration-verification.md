# PROC-EQP-007 試運転・復旧確認

| 状態 | 初版草案 |

> 本文書は修繕、保守、停止・隔離又は異常対応後の標準的な復旧確認を定める。設備別のメーカー手順、保安規程、施工計画、試験要領、法定検査及び操作票を優先する。本手順による技術的復旧確認だけで施設利用再開を決定してはならない。

## 1. 目的

作業後の設備について、作業完了記録、復旧前条件、試運転、保護機能、関連設備及び一定時間の安定性を確認し、技術的に復旧した状態、条件付き運転、復旧保留又は復旧不可を判定する。その結果と残留リスクを施設運用上の利用再開権限者へ引き渡し、技術的復旧と施設利用再開の責任を分離する。

## 2. 業務カタログとの関係

| 区分 | 業務ID | 業務名 | 本手順との関係 |
|---|---|---|---|
| 主業務 | `BM-10-10` | 完了検査・引渡しを行う | 動作・品質・安全を確認し運用側へ引き渡す |
| 関連業務 | `BM-10-09` | 修繕・工事を実施する | 施工完了・自主検査・変更内容を受領する |
| 関連業務 | `BM-10-11` | 修繕履歴を登録する | 原因、処置、部品、試験及び残留事項を履歴化する |
| 関連業務 | `BM-08-02`、`BM-08-06` | 起動・停止、運転状態記録 | 承認済み操作と試運転値を記録する |
| 関連業務 | `BM-13-02`〜`BM-13-06` | 作業記録、確認、承認 | 試験結果と判定・引渡しを記録する |
| 関連業務 | `BM-14-02`、`BM-14-08` | 設備台帳・履歴管理 | 設定、構成、図面及び履歴を更新する |

## 3. 適用範囲

### 3.1 適用対象

- 修繕、部品交換、調整、清掃・給油等の保守後
- 警報・異常の一次対応、緊急停止、隔離又は応急処置後
- 停電・断水・運転停止後の再起動及び系統復帰
- 工事・設定変更・系統切替え後の完了確認
- 専門業者からBM・施設運用側への技術的引渡し

### 3.2 適用除外・禁止事項

- 資格、操作権限、メーカー手順又は必要な検査・承認を満たさない試運転
- 工具、仮設、ロックアウト、接地、バイパス等の状態が不明な再起動
- 防護・保護機能を無断で無効化した運転
- 原因不明・危険継続状態で、警報が消えたことだけを根拠とする復旧判定
- 技術担当者が営業、診療、生産、入居等の施設利用再開を単独決定すること

## 4. 実施体制

| 役割 | 主な責任 |
|---|---|
| 作業実施者・専門業者 | 作業内容、変更、未完了、試験要領及び自主検査結果を申告する |
| 試運転実施者 | 資格・操作権限の範囲で起動・切替え・測定を行う |
| 技術確認者 | 試運転結果、保護機能、関連系統及び残留リスクを評価する |
| BM・元請責任者 | 契約範囲の完了確認、証跡受領、異常案件・修繕履歴への接続を行う |
| 施設運用責任者 | 利用影響、周知、段階再開、制限継続及び利用再開を判断する |
| 警備・防災担当 | 区域、入室、避難、火災監視及び利用者誘導を調整する |

作業実施者の自主検査、技術確認者の復旧判定、元請の検収及び施設側の利用再開は別の確認として記録する。

## 5. 開始条件

- [ ] 対象設備・系統・関連設備・重要負荷を特定した
- [ ] 作業内容、交換部品、設定変更、仮設、未完了、施工者の自主検査を確認した
- [ ] 最新のメーカー手順、系統図、操作票、試験要領及び正常基準を確認した
- [ ] 試運転者の資格・操作権限、必要人数、PPE及び通信を確認した
- [ ] ロックアウト、接地、バルブ、バイパス、保護機能等の復旧順序を確認した
- [ ] 下流・上流設備、利用者、テナント、警備・防災、関連作業との調整を完了した
- [ ] 失敗時の安全停止、隔離、退避、連絡及び復帰不能時の対応を決定した
- [ ] 作業区域、立入制限及び解除条件を確認した

## 6. 復旧前確認

| 区分 | 確認内容 |
|---|---|
| 作業完了 | 作業範囲、締結、接続、配線・配管、部品、清掃、異物・工具・残置物 |
| 安全状態 | カバー、ガード、絶縁、漏れ、圧力、温度、可燃物、換気、接地 |
| 設定・構成 | 設定値、運転モード、バルブ・スイッチ位置、ソフト・制御変更 |
| 保護・監視 | インターロック、警報、保護継電器、非常停止、監視通信 |
| 系統影響 | 上流・下流、冗長系、代替系、重要負荷、他工事・他操作 |
| 証跡 | 作業記録、試験成績、交換部品、写真、変更承認、未完了一覧 |

不一致、不明又は必要確認未了がある場合は復旧保留とし、推測で補完しない。

## 7. 標準手順

| No. | 実施内容 | 完了基準 |
|---:|---|---|
| 1 | 作業実施者から完了状態、変更、未完了、注意事項を引き継ぐ | 作業結果と責任範囲が明確 |
| 2 | 復旧前確認を現物・記録・図面で照合する | 不一致と仮設状態が明確 |
| 3 | 試運転条件、順序、測定点、合否基準、監視時間、中止条件を共有する | 関係者が復唱できる |
| 4 | 区域、通信、退路、緊急停止・隔離手段を確認する | 失敗時に安全状態へ戻せる |
| 5 | 操作票・メーカー手順に従い段階的に起動・復帰する | 各段階の状態が基準内 |
| 6 | 電流、圧力、温度、流量、振動等を指定条件で測定する | 値・単位・条件・測定点が有効 |
| 7 | 警報、保護、インターロック、非常停止等を必要範囲で確認する | 安全・保護機能が有効 |
| 8 | 関連設備、冗長系、中央監視、利用側末端を確認する | 局所復旧による波及異常がない |
| 9 | 定めた監視時間、負荷及び運転モードで安定性を確認する | 再発・漏れ・異音・異臭・傾向逸脱がない |
| 10 | 技術的復旧判定と残留リスクを記録する | 8章の状態を決定 |
| 11 | 運用責任者へ結果、制限、監視、再停止条件を引き渡す | 受領・利用判断が記録済み |
| 12 | 台帳、図面、設定、修繕履歴及び異常案件を更新する | 変更後状態を追跡できる |

## 8. 技術的復旧判定

| 状態 | 条件 | 標準対応 |
|---|---|---|
| 復旧 | 全必須試験合格、保護機能有効、関連設備正常、残留リスクが承認範囲内 | 運用責任者へ利用再開判断を依頼 |
| 条件付き復旧 | 安全な限定運転が可能で、負荷・時間・監視・期限・責任者が承認済み | 制限を明示して引き渡す |
| 復旧保留 | 確認・試験・資格・証跡不足、結果不安定、追加判断待ち | 停止又は安全状態を維持 |
| 復旧不可 | 危険、再発、保護機能不良、基準不適合、重大な未完了 | 再停止・隔離し異常上申・再修繕へ |

「復旧」は技術的状態の判定であり、「修繕案件完了」「契約上の検収」「区域解除」「施設利用再開」と同義ではない。

## 9. 施設利用再開への引渡し

運用責任者へ少なくとも次を伝え、受領と判断を記録する。

- 技術的復旧判定、判定者、判定時刻
- 試運転条件、負荷、測定結果、監視時間
- 原因・処置、原因未確定事項、再発可能性
- 条件付き運転の制限、期限、監視・再確認条件
- 異常時の再停止条件、連絡先役割、代替運用
- 利用者、テナント、診療・生産・営業等への残る影響
- 区域解除、周知、顧客承認及び法定報告の状態

運用責任者は、技術結果に加えて利用影響、代替手段、周知、リスク及び権限を確認し、全面再開、段階再開、制限継続又は再開保留を判断する。

## 10. 異常時対応

| 事象 | 初動 | 後続 |
|---|---|---|
| 起動不能・再停止 | 操作反復を避け、安全状態へ戻す | 原因確認、異常上申、再修繕 |
| 警報再発・値逸脱 | 試運転を中止し影響を確認 | 停止・隔離、時系列・条件を記録 |
| 漏れ・異音・異臭・過熱 | 接近・運転を中止し区域管理 | 専門責任者へ引渡し |
| 保護・監視機能不良 | 通常復旧としない | 代替監視・停止を権限者判断 |
| 関連設備・利用側異常 | 当該設備だけで復旧判定しない | 系統・運用影響を再評価 |
| 記録・設定不一致 | 復旧保留 | 変更承認、図面、台帳を照合 |

異常時は[`PROC-COM-004` 異常上申](../00_common/PROC-COM-004_abnormality-escalation.md)へ接続する。

## 11. 権限

| 判断・行為 | 権限 |
|---|---|
| 復旧前確認・承認済み試運転の実施 | L0又は物件別操作権限 |
| 試運転中止・安全停止 | 切迫時の緊急権限又は操作権限 |
| 条件付き復旧、臨時運転、代替監視 | L2 |
| 再修繕、追加費用、長時間制限 | L3 |
| 長期停止、設備更新、重大リスク受容 | L4 |
| 技術的復旧判定 | 資格・専門性・物件別権限を持つ技術確認者 |
| 施設利用再開 | 施設運用権限者 |
| 契約上の検収 | 元請・顧客等の契約権限者 |

停止権、試運転権、技術的復旧判定、区域解除、利用再開及び検収の権限者を個別に記録する。

## 12. 記録・完了条件

- [ ] 対象設備、系統、作業・修繕・異常案件IDを記録した
- [ ] 作業内容、変更、交換部品、仮設、未完了及び自主検査を確認した
- [ ] 操作順序、試験条件、測定値、保護・関連設備確認を記録した
- [ ] 試運転中の異常、中止、再操作及び状態変化を記録した
- [ ] 技術的復旧判定、判定者、時刻、根拠、残留リスクを記録した
- [ ] 運用責任者の受領、利用再開状態、制限、判断者、時刻を記録した
- [ ] 区域状態、検収状態、台帳・図面・履歴更新を記録した
- [ ] 条件付き・未復旧事項に担当、期限、監視・再確認・再停止条件を設定した

本手順は、技術的復旧判定と運用責任者への引渡しが完了し、未解決事項が後続案件で追跡可能な時点で完了できる。施設利用が再開されていなくても、その状態と判断者が明確なら手順完了と異常案件継続を分けて記録する。
