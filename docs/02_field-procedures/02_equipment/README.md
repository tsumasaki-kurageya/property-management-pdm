# 設備現場作業手順

## 1. 目的

本ディレクトリは、設備の運転監視、巡回点検、検針、運転操作、日常保守、警報・異常対応及び復旧確認について、現場で再現可能な標準手順へ分解するための方針を定める。

上位ルールは[ビルメンテナンス現場作業手順](../README.md)に従う。本READMEは設備領域の対象、手順の分割単位、初期手順一覧、固有の判断・権限・記録事項及び作成順を定義する。

## 2. 対象範囲

### 2.1 対象

- 中央監視装置等による運転状態、警報及び傾向の確認
- 電気室、機械室、屋上、設備設置場所等の巡回点検
- 電気、水道、ガス、温度、圧力等の検針・測定
- 承認済み範囲での起動、停止、設定変更及び系統切替え
- 清掃、給油、増締め、消耗品交換等の日常保守
- 警報受信後の情報確認、現場確認、安全確保、一次対応及び上申
- 作業・修繕後の試運転、正常性確認及び利用再開判断者への引渡し

### 2.2 対象外

- メーカー・型式固有の分解、調整、診断又は設定値を扱う詳細手順
- 法定点検・検査、保安規程、資格者業務を一般化した代替手順
- 工事施工、設計変更、設備更新及び長期修繕計画の詳細
- 消防、電気、ボイラー等の専門規程又は緊急計画を置き換える手順

固有手順・法定手順がある場合、本ディレクトリでは参照先、適用条件、一般作業者の境界及び引渡し条件を明示する。

## 3. 手順の分割基準

設備手順は、次のいずれかが大きく異なる場合に分割する。

| 分解軸 | 分割を検討する差 |
|---|---|
| 作業目的 | 監視、点検、測定、操作、保守、異常対応、復旧確認 |
| 対象設備・系統 | 電気、空調、給排水、昇降機、防災等の危険源と影響範囲 |
| 実施場所 | 中央監視、電気室・機械室、屋上、テナント区画等 |
| 必要資格・権限 | 一般作業者、設備担当者、選任者、メーカー、専門業者 |
| 停止・隔離 | 無停止、部分停止、全停止、ロックアウト等の安全措置 |
| 判断基準 | 数値範囲、警報状態、外観・音・振動・臭気、傾向逸脱 |
| 影響範囲 | 単体設備、系統、区画、建物全体、重要業務への影響 |
| 緊急度 | 定常、要観察、異常、緊急停止・避難を要する状態 |

同一目的でも設備種別により危険源、資格、操作及び復旧条件が大きく異なる場合は、共通手順を入口として設備別の下位手順へ分割する。

## 5. 共通手順との接続

| 工程 | 共通手順 | 設備領域での主な確認 |
|---|---|---|
| 勤務開始 | `PROC-COM-001` 勤務・担当引継ぎ | 警報、停止設備、バイパス、工事、継続異常、臨時運転 |
| 作業前 | `PROC-COM-002` 作業前安全確認・危険予知 | 感電、回転体、高温・高圧、漏水、酸欠、高所、第三者影響 |
| 区域設定 | `PROC-COM-003` 作業区域設定・解除 | 機械室等の入室統制、施錠、表示、操作禁止、周知 |
| 異常発生 | `PROC-COM-004` 異常上申 | 影響範囲、一次対応、停止判断、修繕化、緊急連絡 |
| 完了 | `PROC-COM-005` 完了報告（予定） | 運転状態、設定、残留リスク、要観察、未復旧、引継ぎ |

個別設備のメーカー手順、保安規程、系統図及び操作票がある場合は、それらを実行資料として参照し、一般手順で上書きしない。

## 6. 設備固有の記述規則

### 6.1 対象・測定条件

- 設備、系統、設置場所、識別番号及び関連する上流・下流設備を特定する。
- 測定値には単位、測定点、測定器、運転条件、許容範囲、再測定条件及び比較対象を記載する。
- 外観、音、振動、臭気、温度等の定性確認は、観察位置と通常状態を物件別基準で補う。
- 一点の閾値だけでなく、前回値、同条件時、関連測定点及び変化傾向を確認する条件を定める。

### 6.2 判定・異常分岐

| 状態 | 設備領域での例 | 標準対応 |
|---|---|---|
| 正常 | 運転状態・測定値が基準内で、警報・漏れ・異音等がない | 記録して継続・完了 |
| 要観察 | 基準内だが傾向変化、軽微な劣化、再発警報がある | 再測定、期限付き監視、引継ぎ |
| 異常 | 基準外、機能低下、漏れ、異音、警報継続等がある | 安全確保、一次対応、責任者報告、修繕案件化 |
| 危険 | 感電、火災、破裂、重大漏水、設備破損、被害拡大のおそれ | 接近・操作を中止し、退避、停止可能範囲の措置、緊急連絡 |

警報の復帰だけで正常と判定せず、原因、現場状態、関連設備、影響範囲及び再発可能性を確認する。原因未確定又は応急状態の場合は、要観察又は異常として継続管理する。

### 6.3 操作・停止・復旧権限

| 行為 | 標準的な扱い |
|---|---|
| 承認済み操作票どおりの確認・操作 | L0又は物件別に認められたL1 |
| 軽微な設定調整・運転順変更 | 承認済み範囲に限りL1。それ以外はL2 |
| 臨時運転、バイパス、保護機能変更 | 原則L2以上。期限、代替措置、復旧条件を記録 |
| 緊急停止・安全隔離 | 人命・被害拡大防止のための権限を物件別に明示 |
| 復旧・利用再開 | 停止権とは別に、技術確認者と運用決定者を明示 |
| 設備更新・長期停止・リスク受容 | L4。現場手順では決定しない |

作業者が停止できても、再起動できるとは限らない。操作前確認、操作後確認、失敗時の安全状態及び上申条件を各手順へ記載する。

## 7. 記録・証跡

上位READMEの共通記録に加え、次を対象に応じて記録する。

- 設備・系統・測定点の識別情報
- 運転モード、設定値、バルブ・スイッチ等の操作前後状態
- 測定値、単位、測定条件、測定器及び必要な校正情報
- 警報コード、発生・認知・現場到着・復帰時刻
- 外観、漏れ、損傷、表示、計器等の写真
- 異常の影響範囲、一次対応、停止・隔離及び残留リスク
- 原因の確定・未確定、応急・恒久対応の区別
- 試運転結果、技術確認者、利用再開の決定者と時刻
- 要観察項目、期限、次回確認条件及び引継ぎ先

## 8. 適用差分

| 分析軸 | 設備手順へ反映する主な差 |
|---|---|
| 建物用途 | 停止影響、重要負荷、冗長性、利用時間、温湿度・衛生要件 |
| 管理方式 | 常駐監視、遠隔一次確認、巡回時の出動条件、到着目標、携行機材 |
| 契約役割 | 監視者、現場確認者、操作者、専門業者、元請の統制・検収 |
| 法令義務 | 選任者、必要資格、保安規程、法定周期、事故報告、記録保存 |
| 責任分界 | 停止、臨時運転、利用制限、修繕発注、復旧、リスク受容の決定先 |

差分は設備台帳、系統図、メーカー手順、保安規程、運転基準、緊急連絡網及び既存プロファイルへ具体化する。

## 9. 作成順と完了条件

初期作成は次の順序とする。

1. `PROC-EQP-002` 設備巡回点検
2. `PROC-EQP-005` 警報・設備異常対応
3. `PROC-EQP-007` 試運転・復旧確認
4. `PROC-EQP-001` 中央監視・運転状態確認
5. `PROC-EQP-003` 検針・測定値記録
6. `PROC-EQP-004` 設備運転操作・切替え
7. `PROC-EQP-006` 緊急停止・隔離
8. `PROC-EQP-008` 設備日常保守

本領域の初期整理は、各手順について対象設備、主業務ID、開始・完了条件、測定・判定基準、操作権限、異常分岐、停止・復旧の責任、記録及び適用差分が定義され、固有規程へ安全に接続できた状態を完了とする。
