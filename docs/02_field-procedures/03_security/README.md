# 警備現場作業手順

## 1. 目的

本ディレクトリは、警備勤務の開始・交代、入退館管理、鍵管理、巡回、警報監視、不審者・事故対応及び災害初動について、現場で再現可能な標準手順へ分解するための方針を定める。

上位ルールは[ビルメンテナンス現場作業手順](../README.md)に従う。本READMEは警備領域の対象、手順の分割単位、初期手順一覧、固有の判断・連絡・証跡事項及び作成順を定義する。

## 2. 対象範囲

### 2.1 対象

- 勤務開始・交代時の装備、鍵、警報、継続案件及び指示の引継ぎ
- 来訪者、作業者等の受付、本人・訪問先確認、入館証の発行・回収
- 鍵の保管、貸出、返却、照合及び未返却・紛失時の初動
- 指定経路・巡回点における施錠、火気、不審物、設備外観等の確認
- 防犯カメラ、センサー、警報等の監視と現場確認への接続
- 不審者・不審物、負傷、事故、設備異常等の初動、安全確保、連絡、現場保全
- 火災、地震、風水害等における警備担当範囲の初動と避難支援

### 2.2 対象外

- 警察・消防・救急等の公的機関が行う専門活動
- 個別物件の警備計画、消防計画、危機管理計画を置き換える詳細
- 契約・法令・教育で認められていない実力行使、所持品確認、身体拘束等
- 防犯カメラ映像、個人情報及び機密情報の具体的な保存・開示ルール
- 機械警備会社、消防設備、設備担当等の専門手順そのもの

標準手順には、警備員が実施する初動範囲と、施設責任者、設備担当、消防・警察等へ指揮・対応を引き渡す条件を明示する。

## 3. 手順の分割基準

警備手順は、次のいずれかが大きく異なる場合に分割する。

| 分解軸 | 分割を検討する差 |
|---|---|
| 警備目的 | 受付、権限確認、抑止、監視、巡回、初動、避難支援 |
| 対象 | 人物、鍵、入館証、物品、区画、警報、事件・事故、災害 |
| 勤務状態 | 勤務開始、通常勤務、交代、勤務終了、継続案件対応 |
| 場所・時間帯 | 公開区域、制限区域、閉館後、夜間、単独勤務 |
| 事象の緊急度 | 定常、要注意、規則違反、不審、事故、生命危険 |
| 指揮・連絡 | 警備責任者、施設責任者、設備担当、消防、警察、救急 |
| 証跡 | 受付記録、鍵台帳、巡回時刻、映像、現場記録、引継ぎ |
| 権限・安全 | 声掛け、入館拒否、退避、応援要請、現場保全の範囲 |

建物用途や契約体制により経路、連絡先、制限区域が変わる場合は、基本手順を複製せず、警備計画・物件別設定と適用差分で管理する。

緊急時は共通手順の順番を機械的に守るのではなく、人命保護、退避、緊急通報等を優先し、その後に記録・報告を補完する。

## 6. 警備固有の記述規則

### 6.1 本人・権限・事実確認

- 氏名だけで本人と判断せず、物件別に認められた確認方法、訪問先、目的、時間帯及び立入可能範囲を照合する。
- 例外を現場の推測で許可せず、確認不能、予約不一致、期限切れ、権限外等の分岐を定義する。
- 観察事実、関係者の申告及び作業者の推定を記録上で区別する。
- 個人情報・映像・鍵情報は必要最小限に扱い、閲覧、保存、共有及び開示は物件別規程に従う。

### 6.2 判定・初動

| 状態 | 警備領域での例 | 標準対応 |
|---|---|---|
| 正常 | 本人・権限・施錠・巡回点等が基準どおり | 記録して継続・完了 |
| 要観察 | 直ちに危険ではない軽微な不一致、反復事象、死角等 | 再確認、重点監視、期限付き引継ぎ |
| 異常 | 無断立入、鍵未返却、施錠不良、不審物、事故等 | 安全な距離を保ち、応援・責任者連絡、区域管理、記録 |
| 危険 | 暴力、凶器、火災、重大事故、倒壊等の生命危険 | 接近・単独対応を避け、退避、緊急通報、避難支援 |

「異常なし」を記録する場合も、確認した巡回点、時刻及び未確認箇所を明示する。警報復帰、対象者退出又は救急搬送だけで案件を終了せず、現場の引渡し、証跡保全、残課題及び次勤務者への引継ぎを確認する。

### 6.3 権限・安全・引渡し

- L0：受付、照合、巡回、監視、記録等の承認済み定常業務
- L1：安全な範囲での再確認、声掛け、立入保留、応援要請、初期の区域管理
- L2以上：施設の閉鎖、運用変更、特別な利用制限、契約外対応等

身体安全を損なう単独接近・追跡・制止を標準としない。警備員が現場を統制する範囲と、施設責任者、公的機関又は専門担当へ指揮を引き渡す時点を明示する。緊急通報を行う条件は、通常の上申承認と分ける。

## 7. 記録・証跡

上位READMEの共通記録に加え、次を対象に応じて記録する。

- 勤務者、交代時刻、装備・鍵・入館証の数量と照合結果
- 受付・貸出・返却の時刻、確認方法、許可された範囲、承認した役割
- 巡回開始・終了、巡回点、経路変更、未確認箇所と理由
- 警報種別、発生・認知・現場到着・解除時刻及び確認結果
- 人物の外見・行動、発言、車両、物品等の客観的事実
- 事故・災害の場所、負傷・被害状況、安全確保及び区域管理
- 応援、施設責任者、警察・消防・救急等への連絡時刻、内容、回答
- 映像・写真・遺失物等の証跡識別子、保全・引渡し先と時刻
- 未解決事項、重点監視、次回確認条件及び次勤務者への引継ぎ

記録は事実を時系列で残し、後からの推測で上書きしない。訂正方法、閲覧権限及び保存期間は法令、契約、警備計画及び物件別規程に従う。

## 8. 適用差分

| 分析軸 | 警備手順へ反映する主な差 |
|---|---|
| 建物用途 | 公開性、利用者属性、24時間運用、重要区画、群集・イベント、避難支援 |
| 管理方式 | 常駐・巡回・機械警備、単独勤務、駆付け条件、遠隔監視、応援到着時間 |
| 契約役割 | 警備会社、元請BM、施設責任者、設備担当等の指揮・報告・検収経路 |
| 法令義務 | 警備業務、消防・防災、個人情報・映像、記録・教育等の適用条件 |
| 責任分界 | 入館拒否、区域閉鎖、避難、営業継続、情報開示、警察・消防対応の決定先 |

差分は警備計画、消防計画、巡回経路、鍵台帳、入退館規則、緊急連絡網及び既存プロファイルへ具体化する。

## 9. 作成順と完了条件

初期作成は次の順序とする。

1. `PROC-SEC-004` 巡回警備
2. `PROC-SEC-001` 警備勤務開始・交代
3. `PROC-SEC-007` 事故・急病・現場保全
4. `PROC-SEC-002` 入退館受付・確認
5. `PROC-SEC-003` 鍵・入館証管理
6. `PROC-SEC-005` 警報・防犯監視
7. `PROC-SEC-006` 不審者・不審物対応
8. `PROC-SEC-008` 火災・地震等の災害初動

本領域の初期整理は、各手順について主業務ID、開始・完了条件、確認方法、異常・危険の判定、初動範囲、緊急連絡、指揮の引渡し、証跡及び適用差分が定義され、個別の警備計画・消防計画へ接続できた状態を完了とする。
