# ビルオーナー・PM・FM・BM責任分界プロファイル v1.0

## 1. 目的

本書は、ビルメンテナンス業務に関与するビルオーナー、プロパティマネジメント（PM）、ファシリティマネジメント（FM）及びビルメンテナンス（BM）の責任分界を共通言語化する。

責任分界は会社名や契約階層だけでは決まらない。同じ会社が複数機能を兼務する場合や、PMがFM・BMを含めて受託する場合があるため、物件ごとに次を明確にする。

- 誰が目的・方針・要求水準を定めるか
- 誰が予算・投資・リスク受容を決定するか
- 誰が業務を統括し、委託先を管理するか
- 誰が現場で実施し、一次判断するか
- 誰が結果を確認・検収・承認するか
- 誰が法令上の義務を負い、行政へ報告するか
- 平常時と緊急時で、権限がどのように切り替わるか

本書は標準モデルであり、個別案件の契約書、仕様書、法令、管理規約、賃貸借契約、権限規程及び緊急対応計画に優先しない。

## 2. 対象範囲と関連文書

### 2.1 対象範囲

- 建物・設備の運転、点検、清掃、警備及び衛生管理
- 不具合、修繕、設備更新及び中長期修繕計画
- 品質、法令、安全、事故及び災害対応
- 管理予算、発注、検収及び実績報告
- テナント・施設利用者との運用上の調整
- 建物・設備・契約・作業・法定証跡の情報管理

PM固有のリーシング、賃料設定、債権回収、不動産取得・売却、資金調達等は、BMとのインターフェースに限って扱う。オーナーの資産ポートフォリオ運用を代行するAMは必要に応じて記録するが、本書の中心対象とはしない。

## 3. 基本原則

### 3.1 組織と機能を分ける

「PM会社だからPM機能だけを担う」とは限らない。責任分担表には法人名・部署名とは別に、案件上の機能を登録する。

| 登録単位 | 例 |
|---|---|
| 法人・組織 | 所有会社、PM会社、テナント企業、BM会社 |
| 部署・担当 | 資産管理部、物件担当、総務部、設備責任者 |
| 案件機能 | オーナー、PM、FM、BM、専門業者、法定資格者 |
| 権限 | 最終決定、承認、発注、停止、行政報告、検収 |
| 対象範囲 | 建物、共用部、専有部、設備、契約、時間帯 |

### 3.2 責任の種類を分ける

| 責任・権限 | 意味 |
|---|---|
| 方針責任 | 目的、優先順位、許容リスク及び要求水準を定める |
| 最終決定権 | 投資、予算外支出、重大な停止等を最終決定する |
| 統括責任 | 関係者を調整し、期限、品質、費用及び契約履行を管理する |
| 実施責任 | 作業、操作、点検、連絡等を実行する |
| 確認責任 | 成果物や状態が基準を満たすことを確認する |
| 検収・承認権 | 契約上の成果を受領し、次工程・支払等を認める |
| 法的義務 | 法令上、所有者、管理者、占有者等へ課される義務を負う |
| 費用負担 | 点検、修繕、損害、追加作業等の費用を負担する |
| リスク受容 | 残存リスク、延期又は暫定運用を認める |
| 緊急停止権 | 人命・法令・設備保全のため、設備又は利用を直ちに停止する |

RACIは連絡・実行体制の可視化に有用だが、最終決定権、費用負担、法的義務、リスク受容及び緊急停止権を表現し切れない。本書ではこれらを別属性として管理する。

### 3.3 委託しても当然には移転しないもの

- 所有者・管理者等に課される法的義務
- オーナーが留保した投資判断
- 契約に明記されていない費用負担
- 管理権原を伴う判断
- 重大リスクの受容

国土交通省の「建築物の維持保全に関する準則」では、維持保全計画に実施体制、委託、専門技術者の関与及び責任範囲を定める考え方が示されている。したがって「委託済み」ではなく、具体的業務・担当・情報伝達・意思決定方法まで定める必要がある。  
参照：[国土交通省「建築物の維持保全に関する準則」](https://www.mlit.go.jp/notice/noticedata/pdf/201703/00006633.pdf)、[同準則の解説通知](https://www.mlit.go.jp/notice/noticedata/sgml/100/81000117/81000117.html)

## 4. 四つの機能層

| 機能 | 主に最適化する対象 | 主な時間軸 | 代表的な判断 |
|---|---|---|---|
| ビルオーナー | 所有資産の価値、収益、投資及びリスク | 中長期 | 保有方針、投資、予算、重大リスク受容 |
| PM | 個別不動産の収益、賃貸運営及び契約履行 | 年次～月次 | 物件収支、予算執行、テナント・委託先統括 |
| FM | 利用組織の事業活動、利用者価値及び施設性能 | 中長期～日次 | 施設要求、サービス水準、利用条件、事業影響 |
| BM | 建物・設備の状態、現場品質及び作業履行 | 日次～年次 | 運転操作、点検、一次対応、技術提案、記録 |

PMはオーナーに代わり物件の不動産経営を遂行し、キャッシュフローや資産価値の向上を目指す機能として整理できる。  
参照：[国土交通省「プロパティマネジメント業務について」](https://www.mlit.go.jp/singikai/infra/sangyou/hudousan/6/images/05.pdf)

FMは、土地、建物、設備等を経営にとって最適な状態で保有、使用、運営及び維持する総合的な経営活動であり、単なる維持管理より広い。  
参照：[日本ファシリティマネジメント協会「FMとは」](https://jfma.or.jp/whatsFM/index.html)

### 4.1 ビルオーナー

主な責任は次のとおりである。

- 所有目的、保有・売却方針及び資産価値向上方針を定める
- 年間管理予算及び設備投資枠を最終承認する
- 重大な法令・安全・レピュテーションリスクへの対応を決定する
- PM、FM、BM等への委任範囲と権限限度を定める
- 所有者に残る法令上の義務を履行する

### 4.2 PM

主な責任は次のとおりである。

- オーナー方針を物件運営計画及び予算へ展開する
- 物件収支、予算執行、賃貸運営及びテナント関係を統括する
- BM・専門業者の調達、契約、監督、評価及び検収を行う
- 修繕案を収益、予算、賃貸条件及びテナント影響から評価する
- オーナーへ判断材料、リスク及び実績を報告する

### 4.3 FM

主な責任は次のとおりである。

- 利用組織の経営・事業要求を施設要求へ変換する
- 安全性、快適性、生産性、事業継続性及び環境性能の目標を定める
- 利用部門間の要求を調整し、サービス水準を管理する
- 工事、停止、入退去等の利用影響を評価・調整する
- 施設性能と利用実態から改善・更新の優先順位を提案する

### 4.4 BM

主な責任は次のとおりである。

- 契約仕様、運用条件及び法定基準に従って現場業務を実施する
- 建物・設備を監視し、異常を発見・記録・速報する
- 権限範囲内で運転調整、一次対応及び安全確保を行う
- 劣化、故障、品質及び保全に関する技術的所見を提示する
- 作業記録、測定値、写真、点検・修繕履歴等の証跡を作成する

## 5. 所有・利用・管理形態別モデル

国土交通省のBIM活用ガイドラインでも、賃貸ビルと自社ビルでは主体と業務配置が異なることが示されている。  
参照：[国土交通省「BIMを通じた建築データの活用に関するガイドライン Ver.1」](https://www.mlit.go.jp/jutakukentiku/content/001882470.pdf)

### 5.1 投資用・賃貸ビル

| 機能 | 典型的な配置 |
|---|---|
| オーナー | 不動産所有法人・投資家 |
| AM（存在する場合） | ポートフォリオ・投資運用を代行 |
| PM | 物件運営、収支、テナント、BM契約を統括 |
| FM | PMが兼務、又は主要テナント側に別途存在 |
| BM | 共用部・基幹設備等の現場管理を実施 |

### 5.2 自社所有・自社利用ビル

| 機能 | 典型的な配置 |
|---|---|
| オーナー | 自社又はグループ不動産会社 |
| PM | 独立して存在しない場合がある |
| FM | 総務・管財・ファシリティ部門が担当 |
| BM | 社内設備担当又は外部BM会社 |

### 5.3 オーナーと利用者が異なるビル

共用部・基幹設備はオーナー側PM/BM、専有部・利用サービスはテナント側FMが管理することが多い。次の境界を明記する。

- 共用部と専有部
- 躯体・建築設備とテナント内装・テナント設備
- 電気・水・空調等の責任分界点と計量点
- 開館時間、時間外利用及び特別運転
- 原状回復、模様替え及び工事承認
- 法令上の管理権原及び避難・防火上の役割

### 5.4 区分所有・複合用途ビル

区分所有者、管理組合、管理者、PM、用途別FM及びBMが併存し得る。建物全体、共用部、区画及び設備系統ごとに責任を割り当て、全体調整者を一者定める。

### 5.5 一社兼務モデル

同じ法人がPM、FM又はBMを兼務する場合でも、承認者と実施者を役割上は分離する。自己検収や利益相反を許容する範囲、上位承認が必要な条件、監査方法を定める。

## 6. 意思決定レベルと権限委譲

### 6.1 標準レベル

| レベル | 判断例 | 標準的な決定者 |
|---|---|---|
| L0 現場実行 | 定められた手順での点検、操作、記録 | BM |
| L1 現場裁量 | 設定範囲内の運転調整、軽微な一次対応 | BM責任者 |
| L2 運用変更 | 臨時運転、作業順序変更、小修繕 | PM又はFMの委任先 |
| L3 予算・契約変更 | 予算外修繕、契約範囲追加、停止を伴う工事 | PM又はFM、必要に応じオーナー |
| L4 投資・重大リスク | 設備更新、大規模修繕、長期停止、重大リスク受容 | オーナー |

### 6.2 権限閾値

金額だけでなく、次の条件を組み合わせる。

| 閾値属性 | 例 |
|---|---|
| 金額 | 1件、月間、累計、契約残額 |
| 安全 | 人命危険、避難阻害、負傷可能性 |
| 法令 | 違反、報告期限、行政指摘、使用停止 |
| 事業影響 | 全館・区画停止、重要業務停止、復旧目標超過 |
| テナント影響 | 営業停止、補償可能性、複数テナント影響 |
| 資産影響 | 躯体・基幹設備、保証、資産価値、耐用年数 |
| 対外影響 | 消防・警察・行政・報道への連絡 |

### 6.3 権限委譲の必須項目

- 委譲元と委譲先
- 対象業務、対象物件・設備及び時間帯
- 金額・安全・法令・事業影響等の限度
- 事前承認、事後承認又は報告のみの区分
- 再委譲の可否
- 代行順位と不在時の扱い
- 有効期間と見直し日
- 証跡及び監査方法

## 7. 主要業務の標準責任分界

以下の「決定」は最終決定、「統括」は計画・調整・進捗管理、「実施」は現場実行、「確認」は内容・品質確認を示す。個別案件で兼務・委任を上書きする。

| 業務 | オーナー | PM | FM | BM |
|---|---|---|---|---|
| 資産保有・投資方針 | 決定 | 提案 | 利用要求提示 | 状態情報提供 |
| 年間管理予算 | 最終承認 | 策定・執行統括 | 利用要求反映 | 見積・実績提供 |
| 施設要求・サービス水準 | 方針承認 | 契約へ反映 | 定義・評価 | 作業仕様へ具体化 |
| BM契約・委託先選定 | 重要条件承認 | 調達・契約統括 | 要求・評価参加 | 提案・受託 |
| 年間作業計画 | 重要停止等承認 | 契約・予算面確認 | 利用調整 | 作成・実施管理 |
| 日常運転 | 重要方針承認 | 契約監督 | 利用条件提示 | 操作・監視・記録 |
| 品質基準 | 期待水準承認 | 契約基準管理 | 性能・サービス要求 | 作業・自主検査基準化 |
| 法定業務 | 法的義務に応じ決定 | 期限・委託・報告統括 | 利用調整 | 点検・測定・証跡作成 |
| 小修繕 | 権限委譲 | 予算内承認・発注 | 優先度・利用影響確認 | 発見・提案・実施 |
| 設備更新・大規模修繕 | 投資決定 | 収支・発注統括 | 性能・利用要件提示 | 劣化・技術情報提供 |
| 緊急対応 | 重大判断・リスク受容 | 対外・契約調整 | 事業継続・利用者調整 | 安全確保・停止・速報 |
| テナント対応 | 重要方針 | 賃貸・契約対応 | 利用部門調整 | 現場受付・状況報告 |
| 作業結果 | 経営判断に利用 | 契約検収 | 性能・利用影響確認 | 自主確認・報告 |
| 台帳・履歴 | 保有方針・引継ぎ要求 | 情報統括 | 利用情報管理 | 現場情報更新 |

## 8. 日常運転・点検の分界

### 8.1 運転条件

1. オーナーが省エネルギー、資産保全、リスク等の基本方針を示す。
2. FMが営業時間、温湿度、重要区画、事業継続等の利用要求を定義する。
3. PMが賃貸条件、管理予算及び契約仕様との整合を確認する。
4. BMが設備能力、法令、保全条件を踏まえて運転条件案へ具体化する。
5. 権限者が承認し、BMが設定、監視、調整及び記録を行う。

BMは承認済み範囲内で調整できるが、設備能力外、法令違反、安全余裕の低下又は契約水準変更を伴う条件は上申する。

### 8.2 点検結果

| 確認段階 | 主体 | 確認内容 |
|---|---|---|
| 作業者確認 | BM作業者 | 項目完了、測定値、写真、異常有無 |
| 技術確認 | BM責任者・資格者 | 判定、手順、技術的妥当性 |
| 契約確認 | PM又は発注者 | 契約範囲、期限、成果物、未実施 |
| 性能確認 | FM | 利用要求、サービス水準、事業影響 |
| 経営判断 | オーナー | 重大リスク、投資、延期・受容 |

「作業完了」「技術確認済み」「契約検収済み」「是正完了」は別状態で管理する。

## 9. 不具合・修繕・設備更新の分界

### 9.1 標準フロー

1. BMが異常を発見又は受付する。
2. BMが安全確保、応急処置、影響範囲確認及び速報を行う。
3. BMが原因仮説、修繕案、概算、停止条件及び放置リスクを提示する。
4. FMが事業・利用者・サービスへの影響と要求性能を評価する。
5. PMが予算、契約、収益、テナント及び発注条件を評価する。
6. 権限限度に応じてPM、FM又はオーナーが実施・延期・代替策を決定する。
7. PM又はFMが発注・関係者調整を統括する。
8. BM又は工事会社が修繕を実施し、BMが設備状態を確認する。
9. FMが性能・利用回復を、PMが契約成果を確認する。
10. オーナーが必要に応じて投資効果と残存リスクを評価する。

### 9.2 提案と決定を分ける

中長期修繕計画は、次の成果物を分ける。

| 成果物 | 主担当 |
|---|---|
| 劣化・故障・残存寿命の技術評価 | BM、専門業者 |
| 必要性能・利用制約・事業優先度 | FM |
| 工事時期、テナント影響、物件収支 | PM |
| 投資枠、実施年度、リスク受容 | オーナー |

BMが計画書を作成しても、投資判断まで担ったことにはならない。

### 9.3 延期・見送り

延期・見送りには、決定者、理由、残存リスク、暫定対策、再判定期限及び監視条件を記録する。安全又は法令に反する延期は権限委譲の対象にしない。

## 10. 法定業務の分界

法定業務は、契約上の実施者と法的義務主体を分ける。詳細は[法令義務プロファイル](./statutory-duty-profiles.md)を参照する。

| 役割 | 管理内容 |
|---|---|
| 法的義務主体 | 法令上の所有者、管理者、占有者、設置者等 |
| 適用判定責任者 | 物件・設備属性から義務の適否を確認する者 |
| 選任・届出者 | 管理者・主任技術者等の選任及び届出を行う者 |
| 実施統括者 | 期限、資格者、委託、作業停止等を管理する者 |
| 実施者・資格者 | 法定点検、測定、検査等を行う者 |
| 結果確認者 | 不適合、未実施、報告内容を確認する者 |
| 行政報告者 | 所定名義で行政へ提出し、受理・補正を管理する者 |
| 是正決定者 | 是正方法、期限、費用及び使用制限を決める者 |

PM又はBMへの委託を理由に、オーナー等の法的義務が消滅したものとして扱わない。管理台帳には根拠法令、義務主体、委任範囲及び報告名義を保持する。

## 11. 異常・事故・災害時の分界

### 11.1 初動原則

BMは事前承認を待たず、人命保護、被害拡大防止及び法令上必要な措置を行えるよう、緊急権限を定める。緊急操作後は所定時間内に報告し、復旧・再開は別権限とする。

### 11.2 標準分担

| 対応 | オーナー | PM | FM | BM |
|---|---|---|---|---|
| 人命保護・避難・初期消火 | 方針 | 関係者支援 | 利用者統括 | 発見、通報、初動 |
| 設備緊急停止 | 重要方針 | 連絡・影響調整 | 利用影響判断 | 権限内で即時停止 |
| 被害状況把握 | 統合報告受領 | 物件情報統合 | 人・事業影響統合 | 現場確認・速報 |
| 事業継続判断 | 資産面の重大判断 | 物件・テナント調整 | BCP・利用継続判断 | 復旧見込み提供 |
| 復旧方式 | 投資・重大リスク決定 | 発注・費用統括 | 復旧優先度提示 | 技術案・実施 |
| 利用・運転再開 | 重大案件承認 | 契約・対外調整 | 事業・利用面承認 | 技術的安全確認 |
| 行政・保険・報道 | 最終方針 | 実務統括 | 利用者情報提供 | 事実・証跡提供 |

### 11.3 緊急時権限表の必須項目

- 発動条件と解除条件
- 現場指揮者、対策本部長及び代行順位
- 緊急停止できる設備・区画
- 緊急発注・支出限度
- 避難、閉館、立入禁止及び再開の決定者
- 消防、警察、行政、保険会社、テナントへの連絡者
- 第一報、続報及び確報の期限
- 記録、証拠保全及び対外発表の承認経路

## 12. テナント・施設利用者対応

| 問い合わせ・要求 | 主な窓口 | 判断・連携 |
|---|---|---|
| 賃貸借条件、賃料、契約区画 | PM | オーナー承認条件に従う |
| 温熱、照明、清掃等の利用環境 | FM又はPM | BMが現場確認・対応 |
| 故障、漏水、停電等 | BM又は統合窓口 | 緊急度によりPM・FMへ上申 |
| 専有部工事・模様替え | PM | FM要求、BM技術確認、オーナー基準 |
| 苦情・サービス要求 | PM又はFM | BM実績と原因を確認 |
| 停止・点検・工事周知 | PM又はFM | BMが作業情報を提供 |

受付者、回答責任者、承認者を分け、BMが契約・補償・責任の確定回答を独断で行わないようにする。

## 13. 情報・台帳・証跡の管理責任

### 13.1 情報責任の区分

| 区分 | 意味 |
|---|---|
| 情報オーナー | 保存方針、利用目的、アクセス及び引継ぎを決定する |
| 登録者 | 現場・業務で発生した情報を入力する |
| 内容確認者 | 正確性、完全性及び基準適合を確認する |
| 保管管理者 | 版、保存期間、アクセス、バックアップを管理する |
| 利用者 | 権限内で意思決定・業務に使用する |

### 13.2 代表的な情報

| 情報 | 標準的な情報オーナー | 主な登録者 |
|---|---|---|
| 所有・投資・資産情報 | オーナー | オーナー、AM、PM |
| 賃貸借・テナント情報 | オーナー又はPM | PM |
| 施設要求・利用条件 | FM | FM、利用部門 |
| 建物・設備台帳 | オーナー又は管理権原者 | PM、FM、BM |
| 運転・点検・修繕履歴 | オーナー又は発注者 | BM、専門業者 |
| 法定証跡 | 法的義務主体 | PM、BM、資格者 |
| BM契約・検収記録 | 委託者 | PM又はFM |

契約終了、所有者変更、PM/BM交代時には、データ形式、原本、未完了事項、アクセス権、保存義務及び受領確認を含む引継ぎを行う。

### 14.2 カタログへ追加を検討するインターフェース業務

PM・FM固有業務をBMカタログへ広げず、BMとの境界に次の業務候補を置く。

- 資産方針・施設要求・サービス水準を受領する
- 管理予算及び権限限度を受領する
- 運転・保全・品質条件を合意する
- 技術提案、修繕案及び残存リスクを提出する
- 承認・却下・延期及び条件付き承認を受領する
- PM・FM・オーナー別に報告内容を編集・提出する
- 資産計画及び施設計画へ状態・実績情報を提供する
- 契約・管理主体変更時に情報と未完了事項を引き継ぐ

## 17. 合意・レビュー手順

### 17.1 初期設定

1. 契約書、仕様書、管理規約、法令義務及び既存権限規程を収集する。
2. 法人・部署と案件機能を分けて関係者を列挙する。
3. 建物、共用部、専有部及び設備系統の境界を確認する。
4. 主要業務ごとに実施、統括、決定、確認、費用及び法的義務を割り当てる。
5. 金額・安全・法令・事業影響の権限限度を定める。
6. 平常時、異常時、災害時及び復旧時の経路を確認する。
7. 全当事者で合意し、契約・仕様・運用手順へ反映する。

### 17.2 見直しトリガー

- 所有者、AM、PM、FM又はBMの変更
- 契約更新、仕様変更又は組織改編
- テナント入退去、用途変更又は管理権原変更
- 大規模修繕、設備更新又は新設備導入
- 法令改正、行政指摘又は監査指摘
- 事故、災害、重大不具合又はSLA逸脱
- 権限超過、承認遅延、二重指示又は責任空白の発生
