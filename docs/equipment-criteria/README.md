# 設備別点検・判定基準

## 1. 目的

本ディレクトリは、設備共通の現場作業手順へ、設備種別ごとの確認点、状態判定、操作範囲、停止・復旧条件を与える。

[設備現場作業手順](../02_field-procedures/02_equipment/README.md)が「どの順序で実施し、異常時に誰へつなぐか」を定義するのに対し、設備別基準は「その設備の何を見て、どの状態と判定するか」を定義する。今回の実施結果や測定値は[チェックリスト](../03_checklists/02_equipment/)へ記録する。

```text
業務カタログ
  └─ 現場作業手順 PROC-EQP
       └─ 設備別基準 EQC
            └─ 実行記録 CHK-EQP
```

## 2. 3層の責任分担

| 層 | 正とする内容 | 含めない内容 |
|---|---|---|
| 現場作業手順 `PROC-EQP-*` | 作業順序、共通安全ゲート、異常分岐、権限、引継ぎ | 設備ごとの点検項目一覧、型式固有値 |
| 設備別基準 `EQC-*` | 点検点、正常・要観察・異常・危険、操作境界、停止・復旧条件 | 個別物件の設備ID、今回の測定結果 |
| 実行記録 `CHK-EQP-*` | 実施日時、対象設備、値、状態、証跡、未実施、引継ぎ | 恒久的な判定基準、手順本文 |

設備台帳、系統図、メーカー手順、保安規程、法定業務マスター及び物件別運転基準は、3層へ適用条件と固有値を与える参照情報である。設備別基準でそれらを上書きしない。

## 3. ID・ファイル規則

- IDは `EQC-001` のように採番し、一度公開したIDを別設備へ再利用しない。
- ファイル名は `<ID>_<英語の短い識別名>.md` とする。
- 設備種別、主な危険源、判定方法又は操作権限が大きく異なる場合に分割する。
- メーカー、型式、設置場所又は物件だけの差では新しいIDを作らず、適用情報として外部参照する。
- 同じ設備でも日常点検と資格者による定期・法定点検の方法が異なる場合は、適用範囲を分ける。

## 4. 必須項目

各設備別基準は[テンプレート](TEMPLATE.md)に従い、少なくとも次を持つ。

1. 対象設備と機能
2. 接続する業務ID、手順ID及び記録
3. 適用範囲・適用除外
4. 点検点と確認方法
5. 正常・要観察・異常・危険の設備別条件
6. 操作可能範囲と禁止事項
7. 停止、安全隔離、技術的復旧及び利用再開の権限
8. 日常・定期・法定点検の境界
9. 固有値・法令・メーカー情報の参照方法
10. 改訂時の影響確認先

## 5. 接続規則

| 共通手順 | 設備別基準が与える内容 |
|---|---|
| `PROC-EQP-001` 中央監視・運転状態確認 | 監視点、状態組合せ、警報時の現場確認条件 |
| `PROC-EQP-002` 設備巡回点検 | 点検位置、確認方法、通常状態、接近禁止条件 |
| `PROC-EQP-003` 検針・測定値記録 | 測定点、単位、運転条件、比較対象、再測定条件 |
| `PROC-EQP-004` 設備運転操作・切替え | 許可された操作、前提条件、インターロック、操作後確認 |
| `PROC-EQP-005` 警報・設備異常対応 | 警報ごとの危険度、一次確認、専門対応への移管条件 |
| `PROC-EQP-006` 緊急停止・隔離 | 接近禁止、停止可能範囲、安全隔離成立条件 |
| `PROC-EQP-007` 試運転・復旧確認 | 試験項目、正常性確認、技術的復旧条件 |
| `PROC-EQP-008` 設備日常保守 | 一般作業者が実施できる清掃・交換・調整の範囲 |

手順側は基準IDを参照し、基準側は接続する手順IDを列挙する。チェックリストは実行時に使用した手順ID、基準ID・版及び設備IDを記録する。

## 6. 判定の共通ルール

| 状態 | 設備別基準で定義すること | 標準的な接続 |
|---|---|---|
| 正常 | 機能、表示、測定値、外観及び周辺状態が適用基準内 | 記録して運転・点検継続 |
| 要観察 | 直ちに機能喪失・危険ではないが、傾向変化又は軽微な劣化がある | 期限、担当、再確認条件を設定 |
| 異常 | 基準外、機能低下、部材損傷又は継続警報がある | 安全確保、上申、修繕・専門点検へ |
| 危険 | 人命、火災、感電、破裂、重大漏水又は被害拡大のおそれがある | 接近・操作中止、退避、緊急停止・連絡 |

固定値を記載する場合は、単位、測定位置、運転条件、許容範囲、根拠、適用開始日及び見直し条件を併記する。メーカー・型式・物件に依存する値は本文へ固定せず、参照先と未取得時の扱いを記載する。

## 7. 日常・定期・法定点検の境界

| 区分 | 設備別基準で扱う範囲 |
|---|---|
| 日常点検 | 運転中の外観、表示、音、振動、臭気、漏れ、承認済み測定 |
| 定期点検 | 停止、分解、試験、校正等を伴う場合の入口条件と専門手順への参照 |
| 法定点検 | 適用法令、義務主体、資格、周期、報告・保存先への参照 |

設備別基準は、資格者の判断、法定方法又はメーカー作業を一般作業へ置き換えない。適用区分が不明な場合は実施せず、設備責任者又は選任者へ確認する。

| 基準ID | 対象 | 主な接続手順 | 代表性 |

| [`EQC-002`](EQC-002_air-handling-unit.md) | 空調機 | `PROC-EQP-001`〜`008` | 温湿度、風量、回転体、ドレン |

これらは記述規則を検証する代表例であり、全設備の基準が整備済みであることを意味しない。

## 9. 更新・追跡規則

- 業務カタログの業務ID変更時は、接続表と対象基準を確認する。
- `PROC-EQP-*` の目的、開始条件、判定又は権限変更時は、接続する全基準を確認する。
- 基準改訂時は、使用中のチェックリスト、物件別基準及び未完了案件への適用版を明示する。
- 設備別基準の作成だけでは、現場作業手順カバレッジのC4をC1又はC2へ変更しない。手順本文の成立を別に確認する。
