# 業務エクスプローラー 画面・操作仕様 v0.1

## 1. 目的

業務エクスプローラーは、ビルメンテナンス業務カタログ、業務ライフサイクル、横断プロセスを、選択した業務を起点に探索するためのインターフェースである。

利用者が一つの業務から、次の三方向へ迷わず移動できることを目的とする。

1. **詳細度の往還**
   - 業務全体
   - 業務領域
   - 個別業務
   - 詳しい手順
2. **同じ詳細度の横断**
   - この前に行う仕事
   - この後に行う仕事
   - 条件によって分かれる仕事
   - 同じ領域に属する仕事
3. **関連へのジャンプ**
   - 関係する業務
   - 必要な記録・情報
   - 作られる記録・成果物
   - 担当者・関係者
   - 法令・基準

本仕様は、Issue #27〜#34で実装する画面、データ、状態管理、レスポンシブ対応、アクセシビリティの共通契約とする。

## 2. 想定利用者

主な利用者は、ビルメンテナンス業務に詳しくないソフトウェア開発者、企画担当者、業務分析担当者である。

利用者は業務IDや業界用語を事前に知っているとは限らない。そのため、業務IDは識別子として表示するが、操作の主語は日本語の業務名とする。

## 3. 対象範囲

### 3.1 対象

- 18業務領域と178個別業務の検索・選択
- 選択業務の上位、前後、下位、主要関連の表示
- 選択業務を維持した表示方法の切替
- 業務説明、必要なもの、作られるもの、関係者、法令・基準の表示
- ライフサイクル上の現在位置の表示
- URLによる選択状態の共有と復元
- PC、タブレット、スマートフォンでの閲覧

### 3.2 対象外

- 全178業務と全関連線を同時に表示する巨大グラフ
- ブラウザ上での業務・関係データの編集
- 利用者ごとの表示状態のサーバー保存
- 認証、共同編集、コメント
- 製品カバレッジ評価
- 顧客・物件固有データの登録

## 4. UI設計原則

### 4.1 選択中の業務を中心にする

画面の主語は常に「現在選択している業務」とする。表示方法を切り替えても、選択中の業務は維持する。

### 4.2 全件を一度に見せない

初期表示は、選択業務と理解に必要な1ホップ先を基本とする。全178業務を一度に描画しない。

### 4.3 配置の方向に意味を持たせる

- 左右: この前・この後の仕事
- 上下: 上位・下位の詳細度
- 周囲: 順序ではない主要な関連

ただし、画面幅によって配置を変更しても、見出しとラベルで意味を維持する。

### 4.4 専門用語を操作ラベルにしない

内部データモデルや分析成果物では専門語を使用できるが、画面の主要ラベルには初学者が意味を推測できる表現を使う。

### 4.5 未登録と該当なしを区別する

関係データが登録されていない場合に、「関係が存在しない」と断定しない。

- 調査・登録されていない: `情報未登録`
- 確認した結果、対象が存在しない: `該当なし`
- 検索条件に一致しない: `見つかりませんでした`

### 4.6 図に依存しない

ノード図を操作できない環境でも、同等の前後・上下・関連情報へテキスト一覧から移動できるようにする。

### 4.7 URLで復元できる

選択業務と表示方法はURLへ反映し、再読み込み、共有、ブラウザの戻る・進むで状態を復元できるようにする。

## 5. 表示語彙

| 分析・実装上の用語 | 画面で使用する表現 |
| --- | --- |
| プロセスビュー | 仕事の流れ |
| 階層ビュー | 全体から見る |
| 関連ビュー | 関係から探す |
| 上位業務 | この仕事が属する分野 |
| 下位手順 | 詳しい手順 |
| 前工程 | この前の仕事 |
| 次工程 | この後の仕事 |
| 条件分岐 | 条件によって分かれる仕事 |
| 入力 | この仕事に必要なもの |
| 成果物・出力 | この仕事で作られるもの |
| 実施主体 | 主に作業する人 |
| 承認主体 | 確認・承認する人 |
| ライフサイクル | 業務全体の流れ |
| ノード | 業務カード、関連カード |
| エッジ | つながり、関係線 |

業務名、法令名、正式な帳票名など、意味を保つために必要な固有語は平易化せず、そのまま表示する。

## 6. 画面全体の構成

PC表示では、3ペインと下部の現在位置表示で構成する。

```text
┌──────────────────────────────────────────────────────────────┐
│ ビルメンテナンス業務エクスプローラー        業務検索        │
├──────────────┬────────────────────────┬────────────────────┤
│              │ 表示切替               │ 選択業務           │
│ 業務を探す   │ 仕事の流れ             │ の詳細             │
│              │ 全体から見る           │                    │
│ 領域ツリー   │ 関係から探す           │ 説明               │
│ 検索結果     │                        │ 必要なもの         │
│              │ 選択業務を中心にした   │ 作られるもの       │
│              │ 局所マップ             │ 関係者             │
│              │                        │ 関連リンク         │
├──────────────┴────────────────────────┴────────────────────┤
│ 業務全体の流れ: 契約 → 立ち上げ → 計画 → 作業 → …          │
└──────────────────────────────────────────────────────────────┘
```

### 6.1 左ペイン: 業務を探す

目的は、業務IDを知らない利用者が目的の業務を見つけることである。

表示項目:

- 業務名・キーワード検索
- 18業務領域
- 展開した領域に属する個別業務
- 選択中の業務
- 検索結果件数

操作:

- 領域を展開・折りたたむ
- 個別業務を選択する
- 検索結果から個別業務を選択する
- 選択中の業務が属する領域を自動的に展開する

初期表示では18領域を中心に表示し、178業務をすべて展開しない。

### 6.2 中央ペイン: 業務マップ

目的は、選択業務の位置、前後、詳細、関連を視覚的に理解し、次の業務へ移動することである。

上部に次の表示切替を置く。

1. `仕事の流れ`
2. `全体から見る`
3. `関係から探す`

表示切替時に選択業務を変更しない。

### 6.3 右ペイン: この仕事について

目的は、図だけでは分からない業務の意味と付随情報を確認することである。

表示項目:

- 業務ID・業務名
- 初学者向けの短い説明
- この仕事に必要なもの
- この仕事で作られるもの
- 主に作業する人
- 確認・承認する人
- その他の関係者
- 関連する法令・基準
- 関係する業務
- 既存の解説ページ・リファレンスへのリンク

情報量の多い項目は折りたたみ可能にする。ただし、業務名と短い説明は常に表示する。

### 6.4 下部: 業務全体の流れ

目的は、選択業務が全体のどの段階に位置するかを示すことである。

初期の表示段階は次を基本とする。

1. 契約
2. 立ち上げ
3. 計画
4. 作業
5. 記録
6. 異常判定
7. 修繕・是正
8. 報告
9. 請求
10. 改善

個別業務が複数段階に属する場合は、主段階を強調し、副段階を補助表示する。段階の定義と所属はIssue #27のデータモデルで確定する。

## 7. 表示方法

## 7.1 仕事の流れ

### 目的

「この仕事の前後に何が起きるか」を理解する。

### 基本配置

```text
この前の仕事 → 選択中の仕事 → この後の仕事
                              ├ 正常時
                              └ 異常時・例外時
```

### 初期表示

- 選択中の業務
- 主要な前工程
- 主要な次工程
- 主要な条件分岐
- 関係線の意味を示す短いラベル

### 追加表示

- 複数の前工程・次工程
- 2ホップ目以降
- 補助的な関係

追加表示は利用者の操作で開く。初期表示に含めない。

### 操作

- 業務カードを選択すると、その業務を新しい中心にする
- 再配置後も、選択したカードへフォーカスを移す
- パン、ズーム、中央へ戻す操作を提供する
- テキスト一覧からも前後業務を選択できる

## 7.2 全体から見る

### 目的

「この仕事が全体のどこに属し、どこまで細かく分けられるか」を理解する。

### 基本配置

```text
              この仕事が属する分野
                        ↑
同じ分野の仕事 ← 選択中の仕事 → 同じ分野の仕事
                        ↓
                   詳しい手順
```

### 初期表示

- 選択業務の上位領域
- 選択業務
- 同一領域の近接する業務
- 登録済みの詳しい手順

同一領域の全業務を一度に表示せず、選択業務の前後または代表的な兄弟業務を優先する。

### 操作

- 上位領域を選択すると、領域の概要と配下業務一覧を表示する
- 個別業務を選択すると、その業務を中心にする
- 詳しい手順を選択すると、手順の詳細を右ペインに表示する
- 個別業務と手順で画面の主語が異なる場合は、パンくずで位置を示す

## 7.3 関係から探す

### 目的

順序や親子関係では表せない、業務上の関連をたどる。

### 関連の分類

- 関係する業務
- 必要な情報・記録
- 作られる記録・成果物
- 担当者・関係者
- 法令・基準
- 横断プロセス

### 初期表示

選択業務と主要関連のみを表示する。関係型ごとの上限を設け、表示しきれない場合は件数と「さらに表示」を出す。

### 操作

- 関係型で絞り込む
- 業務カードを選択して中心業務を移動する
- 業務以外のカードを選択して右ペインに詳細を表示する
- その対象に関係する業務一覧へ移動する

`関連なし` と `情報未登録` を区別する。

## 8. 選択状態と画面遷移

画面内のすべての入口は、共通の選択処理を呼び出す。

| 操作元 | 操作 | 結果 |
| --- | --- | --- |
| 左ペイン | 業務を選択 | 中央、右、下部を同じ業務へ更新する |
| 中央マップ | 業務カードを選択 | その業務を中心に再配置し、全領域を更新する |
| 右ペイン | 関連業務を選択 | その業務へ移動する |
| 下部 | ライフサイクル段階を選択 | 対応する業務一覧または代表業務を表示する |
| 表示切替 | 別の表示方法を選択 | 選択業務を維持したままマップを切り替える |
| URL読込 | 業務ID・表示方法を復元 | 指定状態を初期表示する |
| ブラウザ履歴 | 戻る・進む | 過去の選択業務・表示方法を復元する |

業務以外の関連カードを一時選択しても、中心業務は保持する。右ペインには一時選択対象の詳細と「中心の業務へ戻る」を表示する。

## 9. URL仕様

GitHub Pagesで直接アクセスと再読み込みを安定させるため、MVPではクエリパラメーター形式を採用する。

```text
/property-management-pdm/explorer/?business=BM-09-06&view=flow
/property-management-pdm/explorer/?business=BM-09-06&view=hierarchy
/property-management-pdm/explorer/?business=BM-09-06&view=relations&type=records
```

### 9.1 パラメーター

| 名前 | 必須 | 値 | 意味 |
| --- | --- | --- | --- |
| `business` | 任意 | 業務ID | 選択中の個別業務 |
| `view` | 任意 | `flow` / `hierarchy` / `relations` | 表示方法 |
| `type` | 任意 | 関係型の表示用識別子 | 関係から探す表示の絞り込み |

### 9.2 既定値

- `business` がない場合: エクスプローラーの説明と業務領域一覧を表示する
- `view` がない場合: `flow`
- 存在しない業務ID: エラーで停止せず、該当業務が見つからない旨と検索導線を表示する
- 未知の `view`: `flow` へフォールバックする

### 9.3 履歴

- 業務選択と表示切替はブラウザ履歴へ追加する
- パン、ズーム、ペイン開閉など一時的な見た目は履歴へ追加しない
- URLコピー時は、選択業務と表示方法を含める

## 10. 表示状態

### 10.1 初期未選択

- エクスプローラーでできることを短く説明する
- 左ペインの領域一覧または検索を主導線として示す
- 中央に巨大な全体グラフを表示しない

### 10.2 読み込み中

- 画面全体を空白にせず、シェルと読み込み位置を表示する
- 既存の選択状態がある場合は、可能な範囲で保持する

### 10.3 検索結果なし

- `見つかりませんでした` と表示する
- 入力語の見直しと領域一覧への戻り方を示す

### 10.4 業務ID不明

- 指定された業務IDが存在しないことを表示する
- URLを自動的に別業務へ書き換えない
- 検索・領域一覧から選び直せるようにする

### 10.5 関係データ未登録

- `情報未登録` と表示する
- 関係がないとは断定しない
- データの不足によって推測した線を描画しない

### 10.6 描画エラー

- 図の代わりにテキスト一覧を表示する
- 再試行操作を提供する
- 他のStarlightページまで操作不能にしない

## 11. レスポンシブ仕様

ブレークポイントの最終値は実装時に既存Starlightのレイアウトと調整する。ここでは情報の優先順位を定義する。

### 11.1 PC

目安: 72rem以上。

- 左、中央、右の3ペインを同時表示する
- 下部にライフサイクルを表示する
- 中央マップを最も広くする
- 左・右ペインは折りたたみ可能にする

### 11.2 タブレット

- 中央マップを常時表示する
- 左ペインはドロワーまたは折りたたみパネルにする
- 右ペインは選択時にオーバーレイまたは下部シートで表示する
- ライフサイクルは横スクロール可能にする

### 11.3 スマートフォン

3ペインを横に詰め込まない。

- `業務を探す`、`つながりを見る`、`この仕事について` を段階的に切り替える
- マップにはテキスト一覧への切替を用意する
- ノード図のパン操作とページスクロールが競合しないよう、明示的に全画面表示へ切り替えられるようにする
- ライフサイクルは現在位置を中心とした簡略表示とする
- 主要操作を画面下部へ隠しすぎない

## 12. キーボード操作とアクセシビリティ

### 12.1 共通

- フォーカス位置を視認できる
- 色だけで状態や関係型を区別しない
- アイコンだけのボタンには名前を付ける
- `prefers-reduced-motion` に従い、再配置アニメーションを抑制できる

### 12.2 左ペイン

- 検索欄へ直接移動できる
- ツリーを上下キーで移動できる
- 左右キーまたは明示ボタンで展開・折りたたみできる
- EnterまたはSpaceで選択できる

### 12.3 表示切替

- タブとして操作できる
- 選択中の表示方法を読み上げ可能にする
- 左右キーで切り替えられる

### 12.4 中央マップ

- ノードをキーボードで順に選択できる
- 選択ノードの業務名と関係を読み上げ可能にする
- 関係線そのものを操作必須にしない
- 同等情報をテキスト一覧で提供する

### 12.5 右ペイン

- 見出し構造を保つ
- 折りたたみ項目の開閉状態を通知する
- 関連リンクを通常のリンクまたはボタンとして操作できる

実装時の名称は変更可能だが、責務は次のように分離する。

| 仮称 | 責務 |
| --- | --- |
| `ExplorerShell` | 画面レイアウト、共通状態、レスポンシブ制御 |
| `BusinessSearchTree` | 検索、領域ツリー、業務選択 |
| `BusinessMap` | 表示切替、局所グラフ、ノード操作 |
| `FlowMap` | 前後工程と条件分岐の表示 |
| `HierarchyMap` | 上位、同階層、下位の表示 |
| `RelationsMap` | 型付き関連の表示と絞り込み |
| `BusinessDetailPanel` | 選択業務・関連対象の詳細表示 |
| `LifecycleStrip` | 業務全体の流れと現在位置 |
| `ExplorerState` | 選択状態、表示方法、URL、ブラウザ履歴の同期 |
| `TextRelationshipList` | 図と同等のテキスト表現 |

## 14. データとの境界

画面コードへ代表業務や関係を直書きしない。Issue #27で生成する可視化用データを使用する。

画面が必要とする最小情報:

- ノードID
- ノード種別
- 表示名
- 短い説明
- 上位ID
- 前後関係と分岐条件
- 必要なもの
- 作られるもの
- 関係者
- 法令・基準
- ライフサイクル段階
- 横断プロセス
- 詳細ページURL
- 情報の記載状態

関係には種類を持たせる。画面上で区別できない無型の関連線を増やさない。

## 15. 代表シナリオ

`BM-09-06 点検異常を判定する` を代表シナリオとして使用する。

### 15.1 仕事の流れ

- この前の仕事: `BM-09-05 点検値を記録する`
- 選択中: `BM-09-06 点検異常を判定する`
- 正常時のこの後: `BM-09-10 点検結果を承認する`
- 異常時のこの後: `BM-10-01 不具合を受け付ける`

### 15.2 全体から見る

- この仕事が属する分野: `BM-09 点検・保守管理`
- 同じ分野の仕事: BM-09配下の個別業務
- 詳しい手順: 正本で登録された場合のみ表示

### 15.3 関係から探す

- 点検記録
- 設備・作業履歴
- 判定基準
- 担当者・確認者
- 関係する法令・基準

具体的な関連先は正本データに存在するものだけを表示し、モックの見た目を再現するために推測して追加しない。
