# ビルメンテナンス管理方式プロファイル v1.0

## 1. 目的

本書は、ビルメンテナンス業務における常駐管理、巡回管理、遠隔監視、駆付け対応等の管理方式によって、業務の実施方法がどのように変わるかを整理する。

業務そのものは[ビルメンテナンス業務カタログ](./building-maintenance-business-catalog.md)を共通基盤とし、本書では業務を方式別に複製しない。各共通業務に対し、実施場所、実施時間、検知経路、初動、体制、引継ぎ、証跡等の差分を付与する。

本書は業務分析を目的とする。特定製品の機能、製品カバレッジ、費用対効果の評価は対象外とする。

## 2. 管理方式を独立軸とする理由

[建物用途別ビルメンテナンス業務プロファイル](./building-use-profiles.md)が「何を重視して管理するか」を表すのに対し、管理方式は「誰が、いつ、どこから、どの経路で業務を実行するか」を表す。

建物用途と管理方式は一対一ではない。同じオフィスビルでも常駐、定期巡回、遠隔監視との組合せがあり、同一物件内でも設備、清掃、警備、時間帯によって方式が異なる。したがって、管理方式は次の単位で設定する。

> 物件・区画・設備 × 業務領域 × 時間帯 × 通常時・緊急時

## 3. 用語定義

| 用語 | 定義 | 注意点 |
| --- | --- | --- |
| 常駐管理 | 担当者が契約で定めた時間帯に対象物件を拠点として業務を行う方式 | 24時間配置とは限らない |
| 巡回管理 | 担当者が複数物件を担当し、計画した日時に各物件を訪問する方式 | 建物内の巡回点検とは異なる |
| 遠隔監視 | 中央監視、BEMS、センサー、機械警備等から離れた場所で状態・警報を確認する方式 | 現場でしか検知できない事象を補完できない場合がある |
| 遠隔操作 | 権限と安全条件を確認したうえで、離れた場所から設備の設定変更、起動・停止等を行うこと | 遠隔監視の有無と遠隔操作の可否は分けて定義する |
| 駆付け対応 | 警報・通報・判断に基づき、担当者または協力会社が現地へ向かう対応 | 受付から現地到着までの段階管理が必要 |
| 顧客側日常管理 | 顧客・施設利用者側が日常確認や一次連絡を行い、受託会社が専門業務を担う方式 | 責任分界と判断限界を明示する |
| 現場巡回 | 常駐者または巡回担当者が、建物内の設備・区画を見回って確認する業務 | BM-08-04、BM-11-04等の現場業務を指す |
| オンコール | 担当者が現場外で連絡を受けられる状態を確保し、必要時に判断・手配する体制 | 現地到着を保証するものではない |

常駐者が現場巡回を行うことも、巡回管理担当者が訪問時に現場巡回を行うこともある。「巡回管理方式」と「現場巡回業務」は記録上も区別する。

## 4. 管理方式マスター

| 方式ID | 管理方式 | 基本形 | 主な成立条件 |
| --- | --- | --- | --- |
| MODE-01 | 営業・開館時間内常駐 | 利用時間中に物件内へ配置 | 日中の操作・受付・利用者対応が継続的に発生する |
| MODE-02 | 長時間常駐 | 早朝・夜間を含む複数シフト | 営業時間が長い、前後作業が多い |
| MODE-03 | 24時間常駐 | 交代勤務で常時配置 | 停止影響が大きく、即時の現場判断が必要 |
| MODE-04 | 定期巡回 | 曜日・周期・訪問枠を定めて訪問 | 日常操作が少なく、作業を訪問日に集約できる |
| MODE-05 | 遠隔監視＋定期巡回 | 平常時は遠隔監視し、定期的に現地確認 | 監視可能な設備と現場確認対象が整理されている |
| MODE-06 | 遠隔監視＋駆付け | 警報を遠隔受信し、必要時に現地対応 | 24時間受付、判定基準、到着目標、鍵・入館手段がある |
| MODE-07 | 顧客日常管理＋専門巡回 | 顧客が日常確認し、専門者が定期訪問 | 顧客側の役割・教育・連絡条件が明確である |
| MODE-08 | 複合管理 | 業務・設備・時間帯別に複数方式を組み合わせる | 各方式間の責任分界と引継ぎが定義されている |

MODE-08は独立した単一方式ではなく、MODE-01〜07の合成を表す。

## 5. 方式差を表現する属性

| 属性 | 記録内容 |
| --- | --- |
| 適用対象 | 建物、棟、階、区画、設備、業務領域 |
| 適用時間 | 曜日、時間帯、営業日、休日、特別日 |
| 実施主体 | 常駐者、巡回担当、監視センター、顧客、協力会社等 |
| 実施場所 | 現地、監視拠点、顧客拠点、移動中待機等 |
| 検知経路 | 目視、音、臭い、計測値、警報、電話、利用者申告等 |
| 受付条件 | 受付窓口、受付時間、対象、必要情報 |
| 判断権限 | 遠隔判断、設備停止、応急処置、業者手配等の権限 |
| 初動・到着目標 | 受信、確認、連絡、出動、到着、応急処置の各目標時間 |
| 現地不在時の代替 | 遠隔監視、顧客確認、機械警備、駆付け等 |
| 引継ぎ | シフト間、訪問間、監視拠点から現地、顧客から受託会社への申送り |
| 証跡 | 監視ログ、通話、訪問、位置、写真、測定値、承認、復旧確認等 |
| 例外条件 | 災害、通信断、入館不能、同時多発、担当者不在等 |

「対応可能」とだけ記載せず、受付、判断、出動、到着、処置を分けて記録する。特に巡回・駆付けでは、受付時間と現地到着時間を同一視しない。

## 6. 常駐管理と巡回管理の基本差

| 比較軸 | 常駐管理 | 巡回管理 |
| --- | --- | --- |
| 拠点 | 対象物件内 | 複数物件・地域を担当 |
| 現場滞在 | 契約時間中は継続 | 計画訪問中のみ |
| 日常検知 | 警報に加え、目視・音・臭い・体感・利用者申告 | 訪問時確認、遠隔警報、顧客通報 |
| 初動 | その場で現場確認しやすい | 遠隔判定、出動、移動を経て現場確認 |
| 設備運転 | 状況に応じた継続的な操作・微調整が可能 | 自動運転を基本とし、訪問時に確認・調整 |
| 日常依頼 | 現地窓口として随時受付しやすい | 受付窓口から訪問計画または駆付けへ接続 |
| 人員計画 | 物件単位の配置・シフト | エリア、訪問ルート、移動時間、代替要員を計画 |
| 引継ぎ | シフト交代時の日報・口頭確認 | 訪問記録、未確認箇所、次回までの観察事項 |
| 標準化 | 現場固有知識が蓄積しやすい | 誰が訪問しても再現できる台帳・手順が重要 |
| 緊急時 | 常駐時間内は即時の現場確認が可能 | 受付・判定・出動・到着の体制が重要 |
| 不在時間 | 契約外時間には別方式が必要 | 訪問外時間には別方式が必要 |
| 主な制約 | 欠員、交代、属人化、夜間体制 | 移動、同時発生、入館、鍵、現地情報不足 |

## 7. 管理方式を決定する条件

| 条件 | 確認事項 | 方式への影響例 |
| --- | --- | --- |
| 運用時間 | 開館時間、24時間稼働、休日、繁忙時間 | 長時間・24時間常駐、時間帯別複合化 |
| 停止影響 | 人命、安全、営業、生産、データ、生活への影響 | 重要設備の常駐・遠隔監視・即時駆付け |
| 初動要求 | 何分以内に何を確認・実施するか | 現地常駐の要否、駆付け拠点配置 |
| 設備の自動化 | 自動運転、中央監視、遠隔操作、自己診断 | 遠隔監視と巡回の成立可能性 |
| 現場固有検知 | 異音、異臭、漏れ、汚損、利用者反応 | 現場巡回の頻度・常駐の必要性 |
| 日常業務量 | 操作、検針、小修繕、受付、立会の件数 | 常駐時間・訪問頻度・担当人数 |
| 規模・複雑性 | 面積、棟数、設備数、動線、重要区画 | 常駐配置、複数担当、巡回所要時間 |
| 利用者対応 | 問い合わせ、鍵、入館、苦情、緊急連絡 | 現地窓口または顧客側一次受付 |
| 物件分布 | 拠点間距離、交通、天候、時間帯 | 巡回ルート、到着目標、代替拠点 |
| 入館条件 | 鍵、警備解除、本人確認、夜間入口 | 駆付け・巡回の実行可能性 |
| 資格・教育 | 必要資格、専任・選任、現場教育 | 配置者、訪問者、遠隔判断者の条件 |
| 外部体制 | 協力会社の受付時間、対応地域、在庫 | 駆付け・修繕の多段階手配 |
| 災害条件 | 停電、通信断、道路寸断、同時多発 | 現地自立性、優先順位、代替連絡 |
| 契約・責任分界 | 対象時間、対象設備、実施・手配・承認者 | 方式ごとの役割とサービス境界 |

方式選定は建物用途だけで決めず、上記条件を個別物件で確認する。

## 8. 業務領域別の差分

### 8.1 業務立ち上げ・計画（BM-03〜04）

| 常駐管理 | 巡回・遠隔管理 |
| --- | --- |
| 物件内の勤務場所、シフト、現場内連絡、常駐時間外の体制を定める | 訪問周期、標準滞在時間、ルート、鍵、入館、駐車、遠隔接続を定める |
| シフト交代時の引継ぎ様式を準備する | 訪問者が変わっても再現できる手順書・設備台帳を準備する |
| 日次・時間帯別の定常作業を割り当てる | 訪問日に実施する作業を束ね、未実施時の再計画条件を定める |
| 欠員時の代替配置を定める | 交通遅延、緊急出動、同時多発による訪問変更を管理する |

### 8.2 人員・協力会社（BM-05）

常駐管理では物件単位の必要人数、資格、休憩、交代、欠員補充を設計する。巡回管理では担当エリア、移動時間、訪問可能枠、車両、携行品、鍵、複数物件の優先順位を設計する。

双方とも、通常時間外はオンコール、監視センター、協力会社等へ切り替わる場合がある。`BM-05-04 シフトを作成する`では、勤務シフトだけでなく管理方式の切替時刻と引継ぎ先を記録する。

### 8.3 清掃・衛生（BM-06〜07）

清掃は「終日常駐」ではなく、毎日決められた時間だけ入館する配置も多い。この場合、作業時間中は現地配置であっても、物件全体の管理方式とは分けて記録する。

巡回型では、清掃・衛生状態の劣化を訪問間で検知する主体、消耗品切れ・漏水・害虫等の通報先、臨時訪問の判定条件を定める。測定・写真・未確認箇所を記録し、次回訪問までの経過観察事項を残す。

### 8.4 設備運転・点検（BM-08〜09）

| 業務 | 常駐管理 | 巡回・遠隔管理 |
| --- | --- | --- |
| 運転監視 BM-08-03 | 監視盤と現場を継続確認する | 監視対象、通信状態、警報欠測を遠隔確認する |
| 現場巡回 BM-08-04 | シフト・時間帯ごとに巡回する | 訪問計画へ組み込み、遠隔で分からない事象を重点確認する |
| 検針 BM-08-05 | 定時に読み取り、変化をその場で確認する | 自動収集または訪問時に読み取り、欠測・異常値を補完する |
| 警報対応 BM-08-07 | 警報受信後、直ちに現場確認しやすい | 受信、一次判定、顧客確認、出動、現地確認を分ける |
| 日常点検 BM-09-02 | 運転状態と合わせて日常的に実施する | 必要周期を訪問頻度で満たせるか確認する |
| 定期点検 BM-09-03 | 常駐者の立会・事前準備と連携する | 計画訪問または専門業者訪問として手配する |
| 点検未実施 BM-09-09 | シフト内で再割当する | ルート変更・緊急出動による未実施を検出して再訪問する |

遠隔監視を採用しても、通信断、センサー対象外、警報設定不良、現場でしか分からない異音・異臭・漏れ等を考慮し、現場確認をなくすかどうかは別途判断する。

### 8.5 不具合・緊急対応（BM-10）

巡回・遠隔管理では、`BM-10-01〜03`を次の段階に分解して体制を定義する。

1. 警報・通報を受信する。
2. 発生物件、設備、時刻、申告内容を特定する。
3. 遠隔情報と履歴を確認する。
4. 安全性、停止影響、拡大可能性から緊急度を判定する。
5. 顧客側で実施可能な安全確保・暫定対応を指示する。
6. 現地出動の要否と担当者を決定する。
7. 入館、鍵、交通、必要部品・工具を確認する。
8. 出動・到着見込み・遅延を関係者へ連絡する。
9. 現地確認と応急処置を行う。
10. 復旧、利用制限、継続監視、恒久対応へ引き継ぐ。

常駐管理でも、常駐時間外、担当資格外、同時多発、立入禁止区画等では同じ段階管理が必要になる。

### 8.6 警備・防災（BM-11）

有人警備、機械警備、巡回警備は設備管理の方式と別に設定する。常駐警備では受付、入退館、鍵、現場確認、避難誘導を継続して担いやすい。機械警備・巡回警備では、警報受信、映像等による確認、警察・消防・顧客への連絡、警備員の出動条件を定める。

災害時は通常時の到着目標が成立しない可能性があるため、現地にいる者の役割、顧客の自衛体制、優先物件、通信断時の連絡、道路寸断時の代替を定義する。

### 8.7 顧客対応・報告・台帳（BM-12〜14）

常駐管理では対面・内線等の随時依頼が増えやすいため、口頭依頼も受付記録へ残す。巡回管理では受付窓口、訪問で対応する依頼と即時出動する依頼の判定、次回訪問日までの暫定措置を明確にする。

巡回管理の訪問記録には、少なくとも次を含める。

- 訪問日時、到着・退出時刻、担当者
- 実施範囲と未確認箇所
- 測定値、写真、異常、応急処置
- 顧客からの申送りと回答
- 次回までの経過観察事項
- 現地でのみ実施可能な残作業
- 鍵・貸出品・警備状態の受渡し

## 9. 時間帯別管理体制

同じ物件でも時間帯により方式が変わる。管理境界では、未処理警報、設備の一時設定、作業中箇所、利用制限、鍵、入館者、未完了依頼を引き継ぐ。

| 時間帯例 | 主方式 | 補完方式 | 主な引継ぎ |
| --- | --- | --- | --- |
| 平日8:00〜18:00 | MODE-01 営業時間内常駐 | 専門業者の計画訪問 | 日中依頼、作業状況、設備変更 |
| 平日18:00〜翌8:00 | MODE-06 遠隔監視＋駆付け | 顧客の夜間責任者 | 未処理警報、夜間作業、鍵・入館 |
| 休日 | MODE-05 遠隔監視＋定期巡回 | オンコール | 休日作業、運転スケジュール、翌営業日対応 |
| 長期休館 | MODE-05 | 臨時巡回 | 停止設備、凍結・漏水・防犯、再開手順 |

引継ぎの成立は「記録を送信した時点」ではなく、受け手、対象、時刻、未解決事項が確認できる状態をもって判断する。

## 10. 複合管理方式の組み立て方

### 10.1 設定単位

1. 対象物件を棟、階、区画、設備系統に分ける。
2. 共通業務カタログから対象業務を選ぶ。
3. 曜日・時間帯を通常、夜間、休日、特別日に分ける。
4. 各組合せに主方式と補完方式を割り当てる。
5. 受付、判断、現場作業、承認の主体を割り当てる。
6. 方式が切り替わる境界の引継ぎを定義する。
7. 通信断、欠員、入館不能、同時多発等の例外を定義する。

### 10.2 適用例

| 対象 | 時間帯 | 主方式 | 補完方式 | 主な業務差分 |
| --- | --- | --- | --- | --- |
| 中央監視・設備運転 | 平日日中 | MODE-01 | 専門巡回 | 常駐者が監視・操作し、専門点検を受け入れる |
| 中央監視・設備運転 | 夜間休日 | MODE-06 | 顧客夜間責任者 | 遠隔受信・判定後、必要時に駆付ける |
| 清掃 | 毎日早朝・夕方 | 時間限定の現地配置 | 臨時訪問 | 定時作業外の汚損は受付後に緊急度判定する |
| 法定・専門点検 | 計画日 | 専門業者訪問 | 常駐者または顧客立会 | 資格、入館、停止、報告・承認を事前調整する |
| 警備 | 開館中 | 有人警備 | 機械警備 | 受付、入退館、現場巡回を行う |
| 警備 | 閉館中 | 機械警備＋駆付け | 顧客連絡 | 警報確認、出動、警察・消防連絡を行う |

## 11. 方式間の引継ぎ

引継ぎが必要となる代表的な境界は次のとおりである。

- 常駐シフト間
- 常駐時間から遠隔監視時間への切替
- 監視センターから駆付け担当者への出動指示
- 顧客側日常管理から専門巡回担当者への申送り
- 巡回担当者から協力会社への修繕手配
- 応急処置から恒久修繕への移行

引継ぎ記録には、対象物件・設備、発生時刻、現在状態、実施済み対応、禁止操作、安全条件、次の判断期限、関係者、証跡の所在を含める。

## 12. 例外・障害時の業務

| 例外 | 必要な業務 |
| --- | --- |
| 通信断・監視不能 | 監視不能を異常として検知し、現地確認、代替監視、復旧確認を行う |
| 常駐者欠員 | 代替配置、業務優先順位変更、顧客通知、未実施管理を行う |
| 巡回遅延・訪問不能 | 到着見込みを更新し、代替担当・協力会社・顧客確認を手配する |
| 鍵・入館不備 | 入館不能を記録し、権限者確認、代替鍵、警備連携を行う |
| 同時多発 | 人命・安全・拡大性・停止影響で優先順位を付け、応援を手配する |
| 災害・道路寸断 | 現地自立対応へ切り替え、優先物件と代替連絡経路を適用する |
| 誤警報・情報不足 | 警報抑止を安易に行わず、原因、再現性、監視設定を確認する |

## 14. 個別物件への適用手順

1. 用途、規模、運用時間、利用者、設備、契約・責任分界を確認する。
2. 業務領域、設備・区画、時間帯ごとに必要な現場業務を洗い出す。
3. 異常の検知経路と、現地不在時に検知できない事象を整理する。
4. 初動・到着・復旧の要求と判断権限を確認する。
5. 主方式と補完方式を割り当てる。
6. 通常時の受付、判断、現場作業、報告の主体を割り当てる。
7. 方式切替時の引継ぎと証跡を定義する。
8. 欠員、通信断、入館不能、同時多発、災害等の例外を定義する。
9. 現地調査と試行運用で移動時間、業務量、警報精度、未検知事象を確認する。
10. 運用時間、設備、契約、拠点配置、障害実績が変わったときに再評価する。

## 15. 本版の位置付けと今後の分析課題

本版は管理方式の代表的な差を共通業務へ適用するための基礎プロファイルである。常駐管理を高品質、巡回管理を簡易と一律に位置付けるものではない。必要な検知、判断、初動、現地作業、証跡が成立するよう、複数方式を組み合わせて具体化する。

今後の業務分析課題は次のとおりである。

- 業務・設備別の標準初動時間と、契約上の目標時間の整理
- 巡回ルート変更、緊急出動、同時多発時の優先順位付け
- 遠隔監視で取得可能な情報と、現場でしか確認できない情報の分類
- 顧客側日常管理へ委ねられる作業と、教育・責任分界の整理
- 元請、再委託先、専門業者、警備会社、監視センター間の役割差
- 災害時の現地自立運用と複数物件の事業継続体制
