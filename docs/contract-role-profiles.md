# ビルメンテナンス契約役割プロファイル v1.0

## 1. 目的

本書は、[ビルメンテナンス業務カタログ](./building-maintenance-business-catalog.md)に整理した共通業務について、委託者、元請け、再委託先等の契約上の立場により、実施、統括、指示、確認、承認、報告の役割がどのように変化するかを分析するための補助カタログである。

契約役割ごとに共通業務を複製せず、共通業務IDを維持したまま、契約ごとの責任分界と情報・成果物の受渡しを記述する。

本書は特定の製品・システムの機能やカバレッジを評価するものではない。

## 2. 対象範囲と前提

- 対象は、ビルメンテナンス会社が受託し、または他社へ再委託し得る建物管理業務である。
- 本書の役割は会社の固定属性ではなく、契約または発注単位の立場を表す。同じ会社が契約Aでは元請け、契約Bでは再委託先となり得る。
- 契約名称だけでなく、実際の業務遂行、指揮命令、費用負担、責任の実態を確認する。
- 再委託しても、元契約上の責任や法令上の義務が当然に移転するとは限らない。
- 法的義務主体、資格者、届出者、元方事業者等は法令・作業・現場条件ごとに異なるため、既存カタログの法定業務マスターと最新法令を個別に確認する。
- 本書は一般的な業務分析モデルであり、個別契約の法的判断を示すものではない。

## 3. 用語定義

| 用語 | 定義 | 留意点 |
| --- | --- | --- |
| 委託者 | 建物管理業務を直接発注する者 | ビルオーナー、PM会社、管理会社、施設運営者等が該当し得る |
| 元請け | 委託者と直接契約し、元契約の履行窓口となる受託者 | 自社施工と再委託を組み合わせ得る |
| 再委託先 | 元請けから元契約業務の一部を受託する事業者 | 元請けとの契約範囲について遂行責任を負う |
| 再々委託先 | 再委託先からさらに一部業務を受託する事業者 | 元契約・再委託契約の制限と承諾条件を確認する |
| 協力会社 | 取引先を広く指す実務上の総称 | 再委託先、メーカー、資材会社、応援会社等を含み得るため契約上の立場を別に記録する |
| 業務責任者 | 契約業務を把握し、相手方との連絡・調整を行う各社の責任者 | 会社間の窓口と社内の指揮命令を区別する |
| 作業責任者 | 担当作業の手順、要員、安全、品質を管理する者 | 作業単位または現場単位で置かれる |
| 作業員 | 所属会社の指揮命令と承認済み手順に従い作業を実施する者 | 発注者側からの日常的な直接指揮を前提にしない |
| 専門事業者 | 資格、登録、許可、メーカー技術等を要する業務を担う事業者 | 再委託先か別途発注先かを明示する |

## 4. 契約階層モデル

標準的な契約・連絡の階層は次のとおりである。

1. 委託者と元請けが元契約を締結する。
2. 元請けが、元契約で認められた範囲を再委託先へ発注する。
3. 再委託先が、認められた場合に限り再々委託先へ発注する。
4. 各社の業務責任者が会社間の要求、報告、変更、承認を接続する。
5. 各社は自社の作業責任者・作業員を自ら配置し、指揮命令する。

階層の深さだけで役割を決めず、業務ごとに次を明示する。

- 誰から何を受託したか
- 何を自社で実施し、何を再委託するか
- 誰が作業方法と要員を決めるか
- 誰が成果を自主確認、検収、受領するか
- 誰が異常時に判断し、誰へ報告するか
- 誰が法令上の義務・届出・選任を担うか

## 5. 責任区分

「責任」を一括して記録せず、少なくとも次の5区分に分ける。

| 責任区分 | 内容 | 標準的な主体 | 再委託時の扱い |
| --- | --- | --- | --- |
| 契約責任 | 相手方との契約を履行し、結果・不適合・変更を説明する | 各契約の受託者 | 元請けの元契約責任と、再委託先の再委託契約責任は別に残る |
| 統括責任 | 複数業務・会社・工程・情報を横断調整する | 元請け | 実作業を再委託しても通常は元請けに残る |
| 作業遂行責任 | 担当範囲を仕様・手順・品質基準どおり実施する | 実施会社 | 再委託先は受託範囲の遂行に責任を持つ |
| 使用者責任 | 自社作業員の配置、労務、教育、指揮命令を行う | 作業員を雇用する会社 | 発注者や元請けが再委託先作業員を直接指揮するものではない |
| 法的責任 | 法令に基づく義務、届出、選任、作業規制等を履行する | 法令・対象ごとに異なる | 契約階層だけでは移転・決定しない |

責任区分とは別に、業務単位で `実施主体`、`指揮命令主体`、`確認主体`、`承認主体`、`報告主体・報告先` を記録する。

## 6. 元請けと再委託先の基本差

| 業務段階 | 元請け | 再委託先 |
| --- | --- | --- |
| 契約 | 委託者要求を元契約の仕様・価格・責任へ具体化する | 再委託仕様、価格、体制、条件を確認して受託する |
| 再委託判断 | 再委託可否を確認し、自社に残す業務と委託範囲を決める | 再々委託の必要性と可否を確認する |
| 体制 | 全体責任者、会社間の分担、連絡・承認経路を構築する | 自社責任者、作業員、資格者、代替要員を配置する |
| 計画 | 物件全体の年間・月間計画を統合し、競合を調整する | 担当範囲の具体的な工程・作業方法を計画する |
| 指示 | 相手方責任者へ契約要求、制約、変更を伝達する | 自社作業員へ作業方法、配置、順序を指揮する |
| 現場作業 | 自社施工部分を実施し、会社間・工種間を調整する | 契約範囲を自ら管理して実施する |
| 品質 | 元契約の品質基準を展開し、成果を確認・検収する | 自主検査し、記録・証跡とともに提出する |
| 安全 | 物件・会社・工種間の共通条件と作業競合を調整する | 自社作業のリスク、教育、保護具、手順を管理する |
| 異常 | 建物全体への影響を判断し、委託者報告と他業務を調整する | 作業停止、安全確保、担当範囲の一次対応、速報を行う |
| 変更 | 委託者と元契約の仕様・価格・期限を調整する | 工数、価格、要員、作業方法への影響を提示する |
| 報告 | 複数社の記録を確認・統合し、委託者へ報告する | 担当作業の結果、異常、未完了、証跡を元請けへ報告する |
| 請求 | 委託者へ元契約分を請求し、再委託分を検収する | 元請けへ出来高・追加作業等を請求する |
| 改善 | 契約・物件全体の傾向を分析し、水平展開する | 担当作業の手順、品質、生産性、安全を改善する |

元請けが自ら現場作業を行う場合、その範囲では元請けが作業遂行主体も兼ねる。再委託先が専門的判断や作業計画を担う場合でも、元請けの統括・検収・委託者説明まで当然に移るものではない。

## 7. 再委託ライフサイクル

### 7.1 再委託開始前

1. 元契約、提案条件、法令から再委託の可否・制限を確認する。
2. 委託者への通知・事前承諾が必要か確認し、必要な手続きを行う。
3. 自社に残す統括・判断・承認業務と再委託範囲を決定する。
4. 必要資格、許可、登録、保険、実績、対応地域、財務・事業継続性等から候補を評価する。
5. 元契約の仕様、品質、安全、報告期限、秘密保持、個人情報、鍵、図面、再々委託制限を再委託条件へ展開する。
6. 責任分担、連絡経路、緊急連絡、承認権限、証跡、検収条件を合意する。
7. 入館・現場教育・アカウント・鍵・必要資料を準備する。

主な関連業務：BM-01-02〜07、BM-02-01〜04、BM-03-03〜08、BM-05-02〜03、BM-05-07〜09、BM-17-05〜09。

### 7.2 実施中

1. 再委託先が担当範囲の作業計画を作成する。
2. 元請けが元契約・全体計画との整合、作業競合、停止・安全条件を確認する。
3. 必要な入館・作業申請と関係者周知を行う。
4. 再委託先が自社の指揮命令下で作業する。
5. 再委託先が進捗、異常、変更、未実施を責任者経由で報告する。
6. 元請けが他業務・委託者への影響を判断し、必要な調整・承認を行う。
7. 両社がそれぞれの責任範囲の品質・安全・証跡を管理する。

主な関連業務：BM-04-01〜10、BM-06〜11、BM-12-08〜10、BM-13-01〜05、BM-17-01〜07。

### 7.3 完了・精算・終了

1. 再委託先が自主検査し、結果、写真、測定値、異常、変更、残作業を提出する。
2. 元請けが再委託仕様に対して検収し、不適合があれば是正を求める。
3. 元請けが複数成果を統合し、委託者へ報告・提出する。
4. 委託者が元契約に基づき成果を受領・検査・承認する。
5. 追加・変更作業の根拠、数量、承認履歴を確認して請求・支払を確定する。
6. 契約終了時は鍵、カード、図面、データ、貸与品を回収し、アカウント・権限を停止する。

主な関連業務：BM-02-07、BM-10-10〜11、BM-13-01〜10、BM-14-06〜10、BM-16-01〜10、BM-17-01〜04。

## 8. 指示・連絡・承認経路

### 8.1 標準経路

会社間の通常連絡は、次の責任者経路を基本とする。

`委託者担当者 → 元請け業務責任者 → 再委託先責任者 → 再委託先作業員`

元請けは再委託先責任者へ契約要求、施設制約、期限、成果条件を伝える。再委託先責任者は、自社作業員の配置、勤務、作業順序、具体的な遂行方法を決定する。

### 8.2 直接連絡を許容する条件

現場では迅速な連絡が必要なため、直接会話を全面禁止するのではなく、内容と権限を区別する。

| 連絡内容 | 取扱い |
| --- | --- |
| 火災、漏電、転落等の差し迫った危険に対する停止・避難要求 | 安全を優先して直接伝達し、直後に責任者へ報告する |
| 施設の入退館、撮影、立入禁止、衛生等の共通ルール | 事前合意したルールとして直接案内できる |
| 契約仕様への不適合、やり直し、追加作業 | 再委託先責任者へ通知し、変更・是正として管理する |
| 作業員個人の配置、勤務時刻、休憩、作業方法の継続的な指示 | 再委託先の指揮命令系統で行う |
| 緊急時の消防・警察・救急への通報 | 契約上の連絡順序より人命・安全確保を優先する |

請負・業務委託と労働者派遣の区分は契約名ではなく実態に即して判断される。運用設計時は、厚生労働省の37号告示と疑義応答集を参照する。

## 9. 品質確認・検収の三段階

再委託を伴う成果確認は、次の三段階に分ける。

| 段階 | 主体 | 確認対象 | 結果 |
| --- | --- | --- | --- |
| 1. 自主検査 | 再委託先 | 作業手順、範囲、測定値、外観、未完了、証跡 | 作業記録・自主検査結果を元請けへ提出 |
| 2. 再委託検収 | 元請け | 再委託仕様、品質、安全、期限、証跡、是正完了 | 受領、差戻し、是正要求、支払判定 |
| 3. 元契約の受領・検査 | 委託者 | 元契約の成果、全体品質、報告内容 | 承認、差戻し、改善・契約対応 |

`BM-09-10 点検結果を承認する`、`BM-13-06 作業結果を承認する`、`BM-13-10 提出・承認状況を管理する`では、どの契約階層の誰が、何に対して承認するのかを記録する。承認は、法令上の義務主体による確認や行政報告を代替しない。

## 10. 異常・事故時の役割分担

| 段階 | 再委託先 | 元請け | 委託者 |
| --- | --- | --- | --- |
| 発見・安全確保 | 作業停止、救護、隔離、設備停止等を権限内で実施 | 他作業・他社への停止や立入制限を調整 | 施設運営上の緊急措置を判断 |
| 速報 | 発生時刻、場所、状態、人的影響、実施済み対応を元請けへ連絡 | 情報を確認し、委託者への速報要否・内容を判断 | 利用者・社内・外部への連絡を判断 |
| 影響確認 | 担当設備・作業・自社要員への影響を確認 | 建物、他設備、テナント、工程、契約への影響を統合 | 営業・事業継続・対外影響を判断 |
| 原因調査 | 技術上・作業上の事実と証跡を収集 | 契約、体制、会社間調整を含め原因を統合 | 必要な調査範囲・対外説明を確認 |
| 復旧 | 承認された範囲で応急・復旧作業を実施 | 復旧方針、追加業者、費用・変更承認を調整 | 停止継続、再開、追加費用等を承認 |
| 再発防止 | 自社手順、教育、機材、確認方法を改善 | 他社・他業務・他物件への水平展開を管理 | 契約・投資・運用上の改善を判断 |
| 正式報告 | 作業記録、写真、時系列、原因、対策を元請けへ提出 | 内容を検証・統合して委託者へ提出 | 報告を受領し、必要な対外報告を行う |

緊急連絡網には通常経路に加え、人命救助、消防・警察・救急、重大な環境流出等の例外経路を定義する。速報と確報を分け、情報が未確定でも安全上必要な連絡を遅らせない。

## 11. 再々委託の管理

再々委託は、再委託先だけの内部事情として扱わない。元請けは少なくとも次を把握する。

- 元契約・再委託契約上の再々委託可否と承諾状況
- 再々委託する業務範囲と理由
- 会社、責任者、作業者、資格・許可・保険
- さらに下位への委託可否
- 指示・報告・緊急連絡の経路
- 元契約要求の展開状況
- 入館権限、鍵、図面、個人情報、機密情報の保有状況
- 自主検査、検収、是正、請求の接続
- 事故・不適合時の調査協力と証跡保存
- 契約終了時の回収・権限停止

階層が増えても、委託者への正式な報告窓口、元請けの統括、各雇用会社の指揮命令を曖昧にしない。

## 13. 既存業務IDとの対応と不足観点

### 13.1 既存業務で表現できる範囲

| 契約役割上の業務 | 主な既存業務ID | 具体化すべき主体 |
| --- | --- | --- |
| 元契約要求・再委託仕様を管理する | BM-01-02〜07、BM-02-01〜04 | 委託者、元請け、再委託先 |
| 全体体制と責任者を構築する | BM-03-04〜05、BM-03-08 | 各社責任者、連絡・承認経路 |
| 再委託先を登録・選定・評価する | BM-05-07〜09 | 元請け、再委託先、再々委託先を区別 |
| 作業を依頼・調整する | BM-04-04〜08 | 契約要求の伝達者と作業指揮者を区別 |
| 作業結果を記録・承認する | BM-13-01〜10、BM-09-10 | 自主承認、元請検収、委託者承認を区別 |
| 不具合・修繕を統括する | BM-10-01〜12 | 発見、判断、実施、費用承認、報告主体 |
| 協力会社請求を検収する | BM-16-07 | どの契約・出来高・証跡に対する検収か |
| 品質・安全・法令を管理する | BM-17-01〜10 | 法的義務主体、統括主体、実施会社 |

### 13.2 現行カタログだけでは曖昧な観点

次は、個別契約プロファイルで必ず補う。将来、共通性と粒度を評価したうえで業務カタログへの追加を検討する。

- 再委託可否・承諾条件を確認する
- 自社に残す業務と再委託範囲を決定する
- 元契約要求を再委託・再々委託条件へ展開する
- 契約階層ごとの責任分担表を作成する
- 再委託先の作業計画と全体計画の整合を確認する
- 責任者間の指示・報告・承認経路を管理する
- 再々委託先と下位委託範囲を把握する
- 契約階層ごとの自主検査・検収・受領を区別する
- 不適合と是正処置を契約階層間で追跡する
- 契約終了時に貸与品・情報・権限を回収する

## 14. 複合的な役割の扱い

一つの物件・会社に単一の役割を割り当てず、契約・業務・対象・期間単位で記録する。

| 例 | 役割の分け方 |
| --- | --- |
| 元請けが設備管理を自社施工し、清掃を再委託 | 設備では元請け兼実施会社、清掃では元請け兼統括・検収主体 |
| 清掃会社が専門洗浄のみ別会社へ依頼 | 清掃会社は元請けに対する再委託先であり、専門洗浄会社に対しては発注者となる |
| オーナーが法定点検会社へ直接発注 | 法定点検会社は元請けの再委託先ではない。元請けとの連携・立会・情報受渡しを別途定義する |
| メーカー保守契約を委託者が保有 | メーカーの作業結果を誰が受領し、設備台帳・月報へ反映するかを定義する |
| 緊急時だけ応援会社を利用 | 発動条件、指示権限、単価、入館、報告、検収を事前定義する |

建物用途プロファイルが「何を重視して管理するか」、管理方式プロファイルが「いつ・どこから実行するか」を表すのに対し、本プロファイルは「誰が誰に対して責任を負い、誰が実施・確認するか」を表す。三軸を個別物件で組み合わせる。

### 15.3 RACIを補助的に使う場合

RACI等の記号を使う場合も、次の情報を文章または項目として併記する。

- `R` が作業実施者か契約履行者か
- `A` が自主承認、検収、最終承認、法令上の確認のどれか
- 指揮命令主体と統括主体
- 報告先と承認期限
- 緊急時に通常経路を越えて実施できる措置

一つの記号へ異なる責任を詰め込まず、契約階層ごとの受渡しが追跡できることを優先する。

## 16. 個別契約への適用手順

1. 委託者、元請け、再委託先、再々委託先、別途発注先を契約単位で特定する。
2. 元契約と各再委託契約の対象、除外、再委託制限、承諾、成果・検収条件を確認する。
3. 共通業務IDごとに、自社実施、再委託、別途発注、対象外を区分する。
4. 契約責任、統括責任、作業遂行責任、使用者責任、法的責任を分ける。
5. 実施、指揮命令、自主確認、検収、最終承認、報告の主体を割り当てる。
6. 元契約要求が下位契約・作業手順・帳票へ不足なく展開されているか確認する。
7. 通常時、変更時、異常・事故時の連絡・判断・承認経路を定義する。
8. 品質確認の三段階と、不適合・是正・再検査の流れを定義する。
9. 情報、鍵、アカウント、貸与品、証跡の授受・保存・回収を定義する。
10. 初回作業、訓練、監査、事故、契約変更、委託先変更時に責任分担を再確認する。

## 17. 本版の位置付けと今後の分析課題

本版は、元請けと再委託先の役割差を共通業務へ適用するための基礎プロファイルである。元請けを管理専業、再委託先を単純な実作業者と一律に位置付けるものではない。契約・業務ごとに複数の責任と実施・確認・承認主体を組み合わせる。

今後の業務分析課題は次のとおりである。

- 清掃、設備、警備、修繕ごとの標準的な責任分担例
- 元契約要求を下位契約・手順・帳票へ展開するトレーサビリティ
- 再委託承諾、資格・許可、保険、反社会的勢力確認等の管理項目
- 再々委託を含む作業者・入館・教育・証跡の一貫管理
- 不適合、事故、追加変更、出来高、請求の契約階層間の接続
- 委託者の直接発注先と元請けの連携モデル
- 法令別の義務主体と、契約上の実施・手配・確認主体の対応
